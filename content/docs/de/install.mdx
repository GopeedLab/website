Die folgenden Methoden funktionieren auf allen Plattformen für den schnellen Download von Gopeed:

1. Besuchen Sie die [offizielle Website](https://gopeed.com), um Gopeed herunterzuladen. Dort wird automatisch die passende Version für Ihr Betriebssystem ausgewählt.
2. Besuchen Sie [GitHub](https://github.com/GopeedLab/gopeed/releases), um die gewünschte Version manuell herunterzuladen.

## Plattformspezifische Installationsanleitung

### Windows

Windows bietet zwei Distributionen: eine portable Version und eine Installationsversion. Die portable Version muss nicht installiert werden — entpacken Sie sie einfach und starten Sie das Programm. Die Installationsversion muss installiert werden, wobei der Installationsprozess dem anderer Software entspricht. Sie können je nach Ihren persönlichen Vorlieben wählen.

> Der Dateiname der portablen Version lautet: Gopeed-v1.x.x-windows-amd64-portable.zip

Alternativ zum direkten Download können Sie Gopeed auch über Paketmanager installieren oder aktualisieren:

- [Scoop](https://github.com/ScoopInstaller/Scoop): Ein von der Community gepflegter Kommandozeilen-Paketmanager für Windows. Er ist leichtgewichtig und flexibel, erfordert keine Administratorrechte für die Installation und macht die Verwaltung und Aktualisierung von Software einfach.

  ```powershell
  scoop bucket add extras
  scoop install extras/gopeed
  ```

- [Winget](https://learn.microsoft.com/de-de/windows/package-manager/winget): Der offizielle Paketmanager von Microsoft, tief in das Windows-Betriebssystem integriert.

  ```powershell
  winget install -e --id monkeyWie.Gopeed
  ```

#### Fehlalarme der Firewall

Im Windows-System können einige Antivirenprogramme Fehlalarme für zwei Kernkomponenten von Gopeed melden: `host.exe` für die Browser-Erweiterungskommunikation und `updater.exe` für Anwendungsaktualisierungen. Um sicherzustellen, dass die Browser-Erweiterung Download-Aufgaben ordnungsgemäß abfangen kann und die Anwendung sich automatisch aktualisiert, fügen Sie bitte das `Gopeed-Installationsverzeichnis` zur Whitelist Ihrer Firewall oder Ihres Antivirenprogramms hinzu.

<Callout type="info">
**Sicherer Download**: Laden Sie Gopeed unbedingt über die in diesem Dokument beschriebenen Methoden herunter — entweder über die [offizielle Website](https://gopeed.com), das [offizielle GitHub-Repository](https://github.com/GopeedLab/gopeed/releases) oder über Scoop bzw. Winget. Vermeiden Sie Drittanbieter-Quellen, um die Sicherheit der Software zu gewährleisten.

**Zu den Fehlalarmen**: Dies kann mit den Eigenschaften von Anwendungen zusammenhängen, die mit der Programmiersprache Go kompiliert wurden. Derzeit gibt es keine gute Lösung dafür. In Zukunft werden wir erwägen, die Komponenten auf Rust-Entwicklung zu migrieren, um dieses Problem zu beheben.

</Callout>

### Linux

Für Linux bieten wir sowohl `.deb`- als auch `.AppImage`-Pakete zur Installation an. Wählen Sie die für Sie passende Variante. Darüber hinaus können Sie auch schnell mit den folgenden Befehlen installieren:

- Flatpak

  ```sh
  flatpak install flathub com.gopeed.Gopeed
  ```

- Snap

  ```sh
  sudo snap install gopeed
  ```

### macOS

Die macOS-Version bietet eine `.dmg`-Datei, die per Doppelklick installiert werden kann. Das Installationspaket unterstützt zwei Architekturen: `Intel` und `Apple Silicon`.

> Tipp: Falls das Öffnen unter macOS fehlschlägt, führen Sie bitte den Befehl `xattr -d com.apple.quarantine /Applications/Gopeed.app` im Terminal aus.

### Android

Die Android-Version bietet eine `.apk`-Datei, die direkt heruntergeladen und installiert werden kann. Sie unterstützt alle CPU-Architekturen.

### iOS

Derzeit wird für die iOS-Plattform nur eine `.ipa`-Datei bereitgestellt, die selbst signiert und installiert werden muss. Es wird empfohlen, [TrollStore](https://github.com/opa334/TrollStore) für die Installation zu verwenden.

> Warum ist es nicht im App Store gelistet?
>
> Aufgrund der Überprüfungsrichtlinien von Apple dürfen Apps, die mit dem BitTorrent-Protokoll zusammenhängen, nicht gelistet werden.
>
> Warum ist es nicht bei TestFlight gelistet?
>
> Weil das Geld fehlt! Das Projekt wird rein aus Leidenschaft betrieben. Wenn ausreichend Sponsoring erhalten wird, wird es sofort bei TestFlight gelistet. Bitte unterstützen Sie uns daher. Open Source ist nicht einfach. Vielen Dank!

### Web

Wenn Sie einen Remote-Download-Service benötigen, können Sie die Web-Version verwenden. Gopeed bietet Web-Versionen für verschiedene Plattformen an. Sie können diese entsprechend Ihrem System und Ihrer CPU-Architektur herunterladen.

![web](/images/guide/install/web.png)

Hier ist ein Beispiel für die Verwendung der Web-Version auf der Windows-Plattform. Die Verwendung auf anderen Plattformen ist ähnlich.

1. Laden Sie die Web-Version herunter. Nach dem Entpacken erhalten Sie einen Ordner. Platzieren Sie diesen an Ihrem gewünschten Ort.
2. Öffnen Sie das Terminal im Hauptverzeichnis des Ordners und führen Sie `./gopeed.exe` aus. Bei erfolgreicher Ausführung sehen Sie folgende Ausgabe:

   ```bash
      _______   ______   .______    _______  _______  _______
    /  _____| /  __  \  |   _  \  |   ____||   ____||       \
    |  |  __  |  |  |  | |  |_)  | |  |__   |  |__   |  .--.  |
    |  | |_ | |  |  |  | |   ___/  |   __|  |   __|  |  |  |  |
    |  |__| | |  `--'  | |  |      |  |____ |  |____ |  '--'  |
    \______|  \______/  | _|      |_______||_______||_______/

    Server start success on http://[::]:9999
   ```

3. Öffnen Sie den Browser und besuchen Sie `http://localhost:9999`.

#### Web-Konfiguration

Die Web-Version unterstützt die Konfiguration über Kommandozeilenparameter oder Konfigurationsdateien. Kommandozeilenparameter können über `./gopeed.exe -h` eingesehen werden:

```sh
$ ./gopeed.exe -h
Usage of C:\Users\levi\Downloads\gopeed-web-v1.3.13-windows-amd64\gopeed.exe:
  -A string
        Bind Address (default "0.0.0.0")
  -P int
        Bind Port (default 9999)
  -T string
        API token, that can only be used when basic authentication is enabled.
  -c string
        Config file path (default "./config.json")
  -d string
        Storage directory
  -p string
        HTTP Basic Auth Password
  -u string
        HTTP Basic Auth Username (default "gopeed")
```

Es wird auch die Konfiguration über eine Konfigurationsdatei unterstützt. Erstellen Sie eine `config.json`-Datei im Hauptverzeichnis mit folgendem Inhalt:

```json
{
  "address": "",
  "port": 0,
  "username": "",
  "password": "",
  "apiToken": "",
  "storageDir": "",
  "whiteDownloadDirs": ["/root/downloads", "/root/dir/*", "/root/dir?abc"]
}
```

Zudem wird die Konfiguration über Umgebungsvariablen unterstützt. Die Regel lautet `GOPEED_ConfigKey`, zum Beispiel:

```sh
export GOPEED_ADDRESS="0.0.0.0"
export GOPEED_PORT="9999"
export GOPEED_USERNAME="gopeed"
export GOPEED_PASSWORD="xxx"
export GOPEED_APITOKEN=""
export GOPEED_STORAGEDIR=""
export GOPEED_WHITEDOWNLOADDIRS="/root/downloads,/root/dir/*,/root/dir?abc"
```

<Callout type="warn">
  Wenn Sie auf einer öffentlichen IP-Adresse bereitstellen, stellen Sie bitte sicher, dass Sie die
  Authentifizierung aktivieren, da andernfalls Sicherheitsrisiken bestehen.
</Callout>

#### Standard-Download-Konfiguration

Wenn Sie die Standard-Download-Konfiguration beim ersten Start des Servers festlegen möchten, können Sie das Feld `downloadConfig` in `config.json` hinzufügen. Konfigurationsdetails finden Sie unter [DownloaderStoreConfig](https://pkg.go.dev/github.com/GopeedLab/gopeed/pkg/base#DownloaderStoreConfig). Hier ist ein Beispiel:

```json
{
  "address": "127.0.0.1",
  "port": 9999,
  "downloadConfig": {
    "downloadDir": "d:/test"
  }
}
```

### Docker

Ein einzelner Befehl:

```sh
docker run --name gopeed -d -p 9999:9999 liwei2633/gopeed
```

Download-Verzeichnis einbinden

```sh
docker run --name gopeed -d -p 9999:9999 \
    -v /path/to/download:/app/Downloads \
    liwei2633/gopeed
```

Datenverzeichnis einbinden

```sh
docker run --name gopeed -d -p 9999:9999 \
    -v /path/to/download:/app/Downloads \
    -v /path/to/storage:/app/storage liwei2633/gopeed
```

Container-Gruppen-ID und Benutzer-ID angeben

```sh
docker run --name gopeed -d -p 9999:9999 \
    -e PGID=100 \
    -e PUID=1000 \
    liwei2633/gopeed
```

Wenn Sie die Authentifizierung aktivieren möchten, können Sie Umgebungsvariablen übergeben:

```sh
docker run --name gopeed -d -p 9999:9999 \
    -e GOPEED_USERNAME="admin" \
    -e GOPEED_PASSWORD="123" \
    -v /path/to/download:/app/Downloads \
    -v /path/to/storage:/app/storage \
    liwei2633/gopeed
```

### Kommandozeile

Die Kommandozeilenversion erfordert eine `Golang`-Umgebung. Wenn Sie noch keine `Golang`-Umgebung installiert haben, können Sie die Installation [hier](https://golang.org/doc/install) nachlesen.

Installationsbefehl:

```sh
go install github.com/GopeedLab/gopeed/cmd/gopeed@latest
```

Nach der Installation können Sie den Befehl `gopeed` zum Herunterladen verwenden. Die genaue Verwendung können Sie mit `gopeed -h` einsehen:

```sh
$ gopeed -h
Usage of gopeed:
  -C int
        Concurrent connections. (default 16)
  -D string
        Store directory. (default "C:\\Users\\levi")
```

#### Beispiele zur Kommandozeilenverwendung

Eine HTTP-Ressource herunterladen:

```sh
gopeed https://example.com/file.zip
```

Eine Torrent-Datei herunterladen:

```sh
gopeed D:/Downloads/file.torrent
```

Einen Magnet-Link herunterladen:

```sh
gopeed magnet:?xt=urn:btih:xxxx
```

Einen ed2k-Link herunterladen:

```sh
gopeed 'ed2k://|file|example.iso|123456789|0123456789ABCDEF0123456789ABCDEF|/'
```
