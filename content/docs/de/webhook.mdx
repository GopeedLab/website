title: Webhook
description: Konfigurieren Sie Webhooks, um automatisch Benachrichtigungen an eine angegebene URL zu senden, wenn Downloads abgeschlossen sind oder fehlschlagen.
---

Gopeed unterstützt die Webhook-Konfiguration und sendet automatisch HTTP-POST-Anfragen an eine angegebene URL, wenn eine Download-Aufgabe abgeschlossen ist oder fehlschlägt. Diese Funktion lässt sich in Benachrichtigungsdienste wie Gotify, Bark, WeCom, DingTalk und Telegram Bot integrieren, um Sie in Echtzeit über den Download-Status zu informieren.

## Aktivierung

1. Öffnen Sie den Gopeed-Client
2. Gehen Sie zu **Einstellungen** -> **Erweitert** -> **Entwickler**
3. Suchen Sie den Abschnitt **Webhook** und klicken Sie auf das Hinzufügen-Symbol
4. Geben Sie die HTTP-Adresse für den Empfang von Benachrichtigungen ein und bestätigen Sie

## Push-Format

Webhooks verwenden HTTP-POST-Anfragen mit `Content-Type: application/json`. Die Struktur des Anfragekörpers sieht wie folgt aus:

```json
{
  "event": "DOWNLOAD_DONE",
  "time": 1712000000000,
  "payload": {
    "task": {
      "id": "task-uuid",
      "protocol": "http",
      "meta": {
        "req": {
          "url": "https://example.com/file.zip"
        },
        "opts": {
          "name": "file.zip",
          "path": "/downloads"
        },
        "res": {
          "size": 104857600,
          "files": [
            { "name": "file.zip", "size": 104857600 }
          ]
        }
      }
    }
  }
}
```

### Ereignistypen

| Ereignis | Beschreibung |
| --- | --- |
| `DOWNLOAD_DONE` | Download-Aufgabe abgeschlossen |
| `DOWNLOAD_ERROR` | Download-Aufgabe fehlgeschlagen |

## Webhook testen

Beim Hinzufügen einer Webhook-URL können Sie auf die Schaltfläche **Testen** klicken. Gopeed sendet dann ein simuliertes `DOWNLOAD_DONE`-Ereignis an die angegebene URL. Der Test gilt nur dann als erfolgreich, wenn der Server HTTP 200 zurückgibt.

## Unterstützung mehrerer URLs

Sie können mehrere Webhook-URLs konfigurieren. Download-Ereignisse werden gleichzeitig an alle konfigurierten Adressen gesendet, was eine einfache Integration in mehrere Benachrichtigungskanäle ermöglicht.

## Häufige Anwendungsfälle

- **Push-Benachrichtigungen**: Integration mit selbst gehosteten Push-Diensten wie Gotify, Bark oder Pushover
- **Instant Messaging**: Senden von Download-Benachrichtigungen über WeCom/DingTalk/Telegram Bot Webhooks
- **Automatisierung**: Auslösen nachgelagerter Aufgaben wie Extraktion, Transcodierung oder Archivierung
