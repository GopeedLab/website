Gopeed soporta el desarrollo de extensiones usando `JavaScript`. Las extensiones pueden mejorar las funcionalidades de Gopeed, como descargar vídeos o música de un sitio web. Puedes aprender más rápidamente a través de los [ejemplos oficiales](https://github.com/GopeedLab/gopeed-extension-samples/blob/main/README_zh-CN.md).

Las extensiones de Gopeed se basan en `git` para lograr una gestión descentralizada de extensiones. Siempre que el código fuente de la extensión esté alojado en un repositorio git remoto, puede ser instalado y actualizado a través de Gopeed. Por lo tanto, ya sea `github`, `gitee`, `gitlab` u otras plataformas de alojamiento git, todas pueden usarse como repositorios de extensiones.

## Inicio rápido

### Usando el andamiaje

Gopeed proporciona un andamiaje para ayudarte a crear rápidamente una plantilla de proyecto de desarrollo de extensiones:

```sh
npx create-gopeed-ext@latest
```

Durante el proceso de creación, verás las siguientes indicaciones:

```sh
√ Project name (gopeed-extension-demo) ...
√ Choose a template » Webpack

Success! Created gopeed-extension-demo at D:\code\study\js\gopeed-extension-demo
Inside that directory, you can run several commands:

  git init
    Initialize git repository

  npm install
    Install dependencies

  npm run dev
    Compiles and hot-reloads for development.

  npm run build
    Compiles and minifies for production.

We suggest that you begin by typing:

  cd gopeed-extension-demo

Happy coding!
```

### Configuración manual

Si no estás familiarizado con las herramientas periféricas de `node.js`, también puedes crear un proyecto manualmente. La estructura de archivos es la siguiente:

```sh
├── index.js
├── manifest.json
```

## Depuración local

Después de construir el proyecto, necesitas hacer una depuración local. Puedes instalar el proyecto de extensión local en Gopeed para depurarlo. Los pasos específicos son los siguientes:

1. Habilita el modo de desarrollador de `Gopeed`. Haz clic en el botón de instalación `5 veces seguidas` en la página de extensiones para habilitar el modo de desarrollador.

![](/images/dev/extension/dev-mode.gif)

2. Haz clic en el botón para seleccionar el directorio de la extensión en el selector de directorios para instalarla.

3. Si usas el modo `webpack` en el andamiaje, puedes iniciar la compilación automática con `npm run dev`.

4. Crea una tarea para ver la extensión en acción.

![](/images/dev/extension/demo.gif)

Se puede ver que la extensión de ejemplo creada a través del andamiaje puede analizar un archivo `example/index.html` al crear una tarea usando el enlace `https://github.com/hello`.

> Nota: El modo de desarrollador solo está disponible en la plataforma de escritorio.

## Explicación del desarrollo

En la sección anterior, pudimos crear una extensión básica y depurarla localmente, pero ¿qué está pasando detrás de escena?

Primero, echemos un vistazo al archivo `manifest.json`, que es el archivo de manifiesto de la extensión. Describe la información de la extensión. Cada proyecto de extensión debe contener un archivo `manifest.json` en el directorio raíz. El archivo de ejemplo de esta sección es el siguiente:

```json
{
  "name": "gopeed-extention-demo",
  "author": "",
  "title": "gopeed extention demo title",
  "description": "gopeed extention demo description",
  "icon": "",
  "version": "1.0.0",
  "homepage": "",
  "repository": {
    "url": ""
  },
  "scripts": [
    {
      "event": "onResolve",
      "match": {
        "urls": ["*://github.com/*"]
      },
      "entry": "dist/index.js"
    }
  ],
  "settings": []
}
```

A continuación, introduzcamos el significado de cada campo uno por uno:

- `name` y `author`: Gopeed usará `<author>@<name>` como el ID de la extensión. Después de completar el campo `author`, se asegura de que no sea sobrescrita por otras extensiones al instalarse, por lo que se recomienda encarecidamente completar el campo `author`.
- `title` y `description`: El título y la descripción de la extensión.
- `icon`: Icono de la extensión, introduce la ruta relativa, por ejemplo: `icon.png`.
- `version`: Versión de la extensión, usando la especificación semver. Cuando la extensión se actualiza, se compara basándose en este campo, así que asegúrate de que el número de versión cumpla con la especificación.
- `homepage`: Página principal de la extensión, por ejemplo: `https://gopeed.com`.
- `repository`: La dirección del repositorio git al que pertenece la extensión. Las extensiones de Gopeed dependen de `git` para lograr una gestión descentralizada de extensiones. Por lo tanto, si tu extensión necesita ser instalada y actualizada por los usuarios, debes alojar el código fuente de la extensión en un repositorio git remoto y configurar este campo.

  Por ejemplo:

  ```json
  {
    "url": "https://github.com/gopeed/gopeed-extension-demo"
  }
  ```

  Si un repositorio git contiene múltiples proyectos de extensiones, puedes especificar un subdirectorio a través del atributo `directory`, por ejemplo:

  ```json
  {
    "url": "https://github.com/GopeedLab/gopeed-extension-samples",
    "directory": "github-contributor-avatars-sample"
  }
  ```

  Al instalar en Gopeed, necesitas usar `#` como separador, por ejemplo `https://github.com/GopeedLab/gopeed-extension-samples#github-contributor-avatars-sample`.

- `scripts`: `¡Atención!` Esta es la configuración del evento de activación de la extensión de Gopeed.

  El evento `onResolve` configurado en el proyecto de ejemplo se activará al analizar tareas. El campo `match.urls` se usa para hacer coincidir la URL creada por la tarea. Si la coincidencia es exitosa, se ejecutará el archivo de script especificado en el campo `entry`.

  > Las reglas de coincidencia son consistentes con las reglas de coincidencia de las extensiones de Chrome, que puedes consultar [aquí](https://developer.chrome.com/docs/extensions/mv3/match_patterns/)

- `settings`: Configuración de la extensión. A través de la declaración de configuración, se puede generar la página de configuración correspondiente en Gopeed para proporcionar ajustes personalizados por el usuario, como `Cookie` personalizado, `User-Agent` personalizado, etc. Por ejemplo:

  ```json
  [
    {
      "name": "cookie",
      "title": "Cookie del sitio web",
      "description": "La cookie se puede obtener a través de las herramientas de desarrollador del navegador",
      "type": "string"
    },
    {
      "name": "quality",
      "title": "Calidad predeterminada",
      "type": "number",
      "value": "1080",
      "options": [
        {
          "label": "1080P",
          "value": "1080"
        },
        {
          "label": "720P",
          "value": "720"
        },
        {
          "label": "480P",
          "value": "480"
        }
      ]
    }
  ]
  ```

  - `name`: Nombre del ajuste, obligatorio.
  - `title`: Título del ajuste, obligatorio.
  - `description`: Descripción del ajuste, opcional.
  - `type`: Tipo del ajuste, valores posibles: `string`, `number`, `boolean`.
  - `value`: Valor predeterminado del ajuste, opcional.

## Escritura de scripts

En la sección anterior, hemos aprendido cómo configurar el manifiesto. Ahora introduzcamos cómo escribir el script de la extensión.

### Entorno de ejecución

El motor de scripts de extensiones de Gopeed está implementado por [goja](https://github.com/dop251/goja), que es un intérprete de JavaScript escrito en Go puro. Sin embargo, dado que goja es solo un entorno de ejecución JS puro, las APIs de `browser` y `node.js` no están soportadas. Actualmente, `Gopeed` implementa las APIs `XMLHttpRequest` y `fetch`, lo que significa que puedes usar estas dos APIs o bibliotecas de terceros basadas en ellas para implementar solicitudes de red, como `axios`, `superagent`, etc.

Otra cosa a tener en cuenta es que goja soporta nativamente la mayoría de la sintaxis `es6+`, pero algunas sintaxis no están soportadas, como `async generator`. Pero no te preocupes, el proyecto creado por el andamiaje ya tiene configurado `babel`, puedes usar la sintaxis ES más reciente, y el script eventualmente será compilado a sintaxis `es5`.

### Explicación del script de demostración

Cuando se activa un evento, se ejecutará el archivo de script especificado en el campo `entry`. El archivo de script en el proyecto de ejemplo es el siguiente:

```js
gopeed.events.onResolve((ctx) => {
  ctx.res = {
    name: "example",
    files: [
      {
        name: "index.html",
        req: {
          url: "https://example.com",
        },
      },
    ],
  };
});
```

Explicación del contenido del script:

- `gopeed.events.onResolve`: Esto registra el controlador del evento `onResolve`, donde reside la lógica de la extensión.
- `ctx`: El contexto del evento, que contiene información sobre el evento actual. En el evento `onResolve`, `ctx` contiene:
  - `req`: Información de la solicitud, incluyendo la URL del recurso, encabezados, etc.
  - `res`: Información de la respuesta. El script debe asignar la lista de archivos analizados a `ctx.res`, y Gopeed descargará según la lista de archivos devuelta.

En resumen, en la función de callback `onResolve`, necesitas analizar la lista de archivos a descargar basándote en la información de la solicitud en `ctx.req` y asignarla a `ctx.res`.

> Para las definiciones de tipos detalladas de las extensiones y el código fuente relacionado, consulta el [repositorio oficial de Gopeed JS](https://github.com/GopeedLab/gopeed-js).
