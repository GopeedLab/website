title: Développement d'extensions
description: Créez des extensions JavaScript pour Gopeed, déboguez-les localement et comprenez le manifeste et l'environnement d'exécution des extensions.
---

Gopeed prend en charge le développement d'extensions en `JavaScript`. Les extensions peuvent améliorer les fonctionnalités de Gopeed, comme télécharger des vidéos ou de la musique depuis un site web. Vous pouvez en apprendre davantage rapidement grâce aux [exemples officiels](https://github.com/GopeedLab/gopeed-extension-samples/blob/main/README_zh-CN.md).

Les extensions Gopeed sont basées sur `git` pour réaliser une gestion d'extensions décentralisée. Tant que le code source de l'extension est hébergé dans un dépôt git distant, il peut être installé et mis à jour via Gopeed. Ainsi, que ce soit `github`, `gitee`, `gitlab` ou d'autres plateformes d'hébergement git, elles peuvent toutes servir de dépôt d'extensions.

## Démarrage rapide

### Utilisation du scaffolding

Gopeed fournit un scaffolding pour vous aider à créer rapidement un modèle de projet de développement d'extension :

```sh
npx create-gopeed-ext@latest
```

Lors du processus de création, vous verrez les invites suivantes :

```sh
√ Project name (gopeed-extension-demo) ...
√ Choose a template » Webpack

Success! Created gopeed-extension-demo at D:\code\study\js\gopeed-extension-demo
Inside that directory, you can run several commands:

  git init
    Initialize git repository

  npm install
    Install dependencies

  npm run dev
    Compiles and hot-reloads for development.

  npm run build
    Compiles and minifies for production.

We suggest that you begin by typing:

  cd gopeed-extension-demo

Happy coding!
```

### Configuration manuelle

Si vous n'êtes pas familier avec les outils de l'écosystème `node.js`, vous pouvez également créer un projet manuellement. La structure des fichiers est la suivante :

```sh
├── index.js
├── manifest.json
```

## Débogage local

Une fois le projet créé, vous devez le déboguer localement. Vous pouvez installer le projet d'extension local dans Gopeed pour le débogage. Les étapes sont les suivantes :

1. Activez le mode développeur de `Gopeed` en cliquant `5 fois de suite` sur le bouton d'installation de la page des extensions.

![](/images/dev/extension/dev-mode.gif)

2. Cliquez sur le bouton pour sélectionner le répertoire de l'extension dans le sélecteur de répertoire pour l'installer.

3. Si vous utilisez le mode `webpack` du scaffolding, vous pouvez lancer la compilation automatique via `npm run dev`.

4. Créez une tâche pour voir l'extension en action.

![](/images/dev/extension/demo.gif)

On peut voir que l'extension d'exemple créée via le scaffolding peut analyser un fichier `example/index.html` lors de la création d'une tâche avec le lien `https://github.com/hello`.

> Remarque : Le mode développeur est uniquement disponible sur la plateforme de bureau.

## Explications sur le développement

Dans la section précédente, nous avons pu créer une extension basique et la déboguer localement, mais que se passe-t-il en coulisses ?

Tout d'abord, examinons le fichier `manifest.json`, qui est le fichier manifeste de l'extension. Il décrit les informations de l'extension. Chaque projet d'extension doit contenir un fichier `manifest.json` dans son répertoire racine. Le fichier d'exemple de cette section est le suivant :

```json
{
  "name": "gopeed-extention-demo",
  "author": "",
  "title": "gopeed extention demo title",
  "description": "gopeed extention demo description",
  "icon": "",
  "version": "1.0.0",
  "homepage": "",
  "repository": {
    "url": ""
  },
  "scripts": [
    {
      "event": "onResolve",
      "match": {
        "urls": ["*://github.com/*"]
      },
      "entry": "dist/index.js"
    }
  ],
  "settings": []
}
```

Introduisons maintenant la signification de chaque champ un par un :

- `name` et `author` : Gopeed utilise `<author>@<name>` comme identifiant de l'extension. Après avoir renseigné `author`, cela garantit que l'extension ne sera pas facilement écrasée par d'autres extensions. Il est donc fortement recommandé de remplir le champ `author`.
- `title` et `description` : Le titre et la description de l'extension.
- `icon` : L'icône de l'extension, renseignez le chemin relatif, par exemple : `icon.png`.
- `version` : La version de l'extension, utilisant la spécification semver. Lors de la mise à jour de l'extension, la comparaison se base sur ce champ. Assurez-vous donc que le numéro de version est conforme à la spécification.
- `homepage` : La page d'accueil de l'extension, par exemple : `https://gopeed.com`.
- `repository` : L'adresse du dépôt git auquel l'extension appartient. Les extensions Gopeed reposent sur `git` pour une gestion d'extensions décentralisée. Par conséquent, si vous souhaitez que votre extension puisse être installée et mise à jour par les utilisateurs, vous devez héberger le code source dans un dépôt git distant et configurer ce champ.

  Par exemple :

  ```json
  {
    "url": "https://github.com/gopeed/gopeed-extension-demo"
  }
  ```

  Si un dépôt git contient plusieurs projets d'extensions, vous pouvez spécifier un sous-répertoire via l'attribut `directory`, par exemple :

  ```json
  {
    "url": "https://github.com/GopeedLab/gopeed-extension-samples",
    "directory": "github-contributor-avatars-sample"
  }
  ```

  Lors de l'installation dans Gopeed, vous devez utiliser `#` comme séparateur, par exemple `https://github.com/GopeedLab/gopeed-extension-samples#github-contributor-avatars-sample`.

- `scripts` : `Attention !` Ceci est la configuration des événements d'activation de l'extension Gopeed.

  L'événement `onResolve` configuré dans le projet d'exemple sera déclenché lors de l'analyse des tâches. Le champ `match.urls` est utilisé pour faire correspondre l'URL créée par la tâche. Si la correspondance réussit, le fichier script spécifié dans le champ `entry` sera exécuté.

  > Les règles de correspondance sont identiques à celles des extensions Chrome. Vous pouvez vous référer à [cette page](https://developer.chrome.com/docs/extensions/mv3/match_patterns/)

- `settings` : Les paramètres de l'extension. Via la déclaration de configuration, une page de paramètres correspondante peut être générée dans Gopeed pour offrir des paramètres personnalisables à l'utilisateur, comme un `Cookie` personnalisé, un `User-Agent` personnalisé, etc. Par exemple :

  ```json
  [
    {
      "name": "cookie",
      "title": "Website Cookie",
      "description": "Cookie can be obtained through browser developer tools",
      "type": "string"
    },
    {
      "name": "quality",
      "title": "Default Quality",
      "type": "number",
      "value": "1080",
      "options": [
        {
          "label": "1080P",
          "value": "1080"
        },
        {
          "label": "720P",
          "value": "720"
        },
        {
          "label": "480P",
          "value": "480"
        }
      ]
    }
  ]
  ```

  - `name` : Nom du paramètre, obligatoire.
  - `title` : Titre du paramètre, obligatoire.
  - `description` : Description du paramètre, facultatif.
  - `type` : Type du paramètre, valeurs possibles : `string`, `number`, `boolean`.
  - `value` : Valeur par défaut du paramètre, facultatif.

## Écriture des scripts

Dans la section précédente, nous avons appris à configurer le manifeste. Voyons maintenant comment écrire le script de l'extension.

### Environnement d'exécution

Le moteur de scripts d'extension de Gopeed est implémenté par [goja](https://github.com/dop251/goja), un interpréteur JavaScript écrit en Go pur. Cependant, comme goja est uniquement un runtime JS pur, les API de `browser` et `node.js` ne sont pas prises en charge. Actuellement, `Gopeed` implémente les API `XMLHttpRequest` et `fetch`, ce qui signifie que vous pouvez utiliser ces deux API ou des bibliothèques tierces basées sur celles-ci pour effectuer des requêtes réseau, comme `axios`, `superagent`, etc.

Autre point important : goja prend nativement en charge la plupart de la syntaxe `es6+`, mais quelques syntaxes ne sont pas prises en charge, comme `async generator`. Ce n'est cependant pas un problème, car le projet créé par le scaffolding est configuré avec `babel`, vous pouvez utiliser la syntaxe ES la plus récente, et le script sera finalement compilé en syntaxe `es5`.

### Explication du script de démonstration

Lorsqu'un événement est déclenché, le fichier script spécifié dans le champ `entry` est exécuté. Le fichier script du projet d'exemple est le suivant :

```js
gopeed.events.onResolve((ctx) => {
  ctx.res = {
    name: "example",
    files: [
      {
        name: "index.html",
        req: {
          url: "https://example.com",
        },
      },
    ],
  };
});
```

Expliquons le contenu du script :

- `gopeed.events.onResolve` : Ceci enregistre le gestionnaire de l'événement `onResolve`, où réside la logique de l'extension.
- `ctx` : Le contexte de l'événement, contenant des informations sur l'événement en cours. Dans l'événement `onResolve`, `ctx` contient :
  - `req` : Informations de la requête, incluant l'URL de la ressource, les en-têtes, etc.
  - `res` : Informations de la réponse. Le script doit assigner la liste des fichiers analysés à `ctx.res`, et Gopeed effectuera le téléchargement en fonction de la liste de fichiers renvoyée.

En résumé, dans la fonction de rappel `onResolve`, vous devez analyser la liste des fichiers à télécharger en fonction des informations de requête dans `ctx.req` et l'assigner à `ctx.res`.

> Pour les définitions de types détaillées des extensions et le code source associé, veuillez consulter le [dépôt officiel Gopeed JS](https://github.com/GopeedLab/gopeed-js).
