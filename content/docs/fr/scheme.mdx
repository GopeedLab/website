Gopeed enregistre le protocole personnalisé `gopeed:`, vous permettant de lancer Gopeed directement depuis un navigateur, une autre application ou la ligne de commande pour effectuer des actions.

## Format URI

```
gopeed:///create?params=<base64-encoded JSON>
gopeed:///extension?params=<base64-encoded JSON>
```

- `<path>` : Type d'action, prend actuellement en charge `/create` et `/extension`
- `params` : Paramètres JSON encodés en Base64

## Créer une tâche de téléchargement

Chemin : `/create`

### Paramètres

| Champ | Type | Description |
| --- | --- | --- |
| `req.url` | string | URL de téléchargement (HTTP, magnet, ed2k, etc.) |

La structure complète des paramètres correspond au corps de la requête de l'API de création de tâche. Consultez la [documentation de l'API de création de tâche](https://gopeed.com/docs/openapi#tag/tasks/POST/api/v1/tasks) pour plus de détails.

### Exemple

JSON des paramètres :

```json
{
  "req": {
    "url": "https://example.com/file.zip"
  }
}
```

## Installer une extension

Chemin : `/extension`

### Paramètres

| Champ | Type | Description |
| --- | --- | --- |
| `url` | string | URL du dépôt Git de l'extension |
| `devMode` | boolean | Indique s'il faut installer en mode développement, par défaut `false` |

### Exemple

JSON des paramètres :

```json
{
  "url": "https://github.com/user/gopeed-extension-example",
  "devMode": false
}
```

## Interception de téléchargement sur Android

Sur Android, Gopeed enregistre un filtre d'intent standard de gestionnaire de téléchargement. Après l'installation, lorsque vous appuyez sur un lien de téléchargement dans un navigateur ou une autre application, le système affiche un sélecteur d'application où vous pouvez choisir Gopeed pour gérer le téléchargement.

Scénarios pris en charge :

- Choisir Gopeed lorsqu'un navigateur affiche une boîte de dialogue de téléchargement
- Choisir Gopeed lors du partage de liens HTTP/HTTPS depuis d'autres applications
- Définir Gopeed comme gestionnaire de téléchargement par défaut dans les paramètres des applications par défaut d'Android

<Callout type="info">
  Après l'installation d'une nouvelle version, Android peut avoir besoin de rescanner les applications pour reconnaître les nouveaux filtres d'intent. Essayez de vider le cache de l'application, de redémarrer l'appareil, ou de réinitialiser les préférences d'application par défaut dans les paramètres système.
</Callout>
