---
title: Exécution de scripts
description: Configurez des scripts hook pour s'exécuter automatiquement après la fin des téléchargements, permettant le déplacement de fichiers, les notifications et d'autres automatisations.
---

Gopeed prend en charge l'exécution automatique de scripts hook spécifiés lorsqu'une tâche de téléchargement **se termine avec succès**. Utilisez cette fonctionnalité pour automatiser le déplacement de fichiers, envoyer des notifications, déclencher un transcodage, et plus encore.

## Comment activer

1. Ouvrez le client Gopeed
2. Accédez à **Paramètres** → **Avancé** → **Développeur**
3. Recherchez la section **Exécution de scripts** et activez le commutateur
4. Ajoutez le chemin complet vers votre fichier script

## Types de scripts pris en charge

Gopeed sélectionne automatiquement l'interpréteur en fonction de l'extension du fichier :

| Extension | Interpréteur | Plateforme |
| --- | --- | --- |
| `.sh` / `.bash` | Bash | macOS / Linux |
| `.py` | Python 3 | Toutes les plateformes |
| `.js` | Node.js | Toutes les plateformes |
| `.bat` / `.cmd` | cmd.exe | Windows uniquement |
| `.ps1` | PowerShell | Toutes les plateformes |
| Sans extension | Exécution directe (nécessite un shebang ou une permission d'exécution) | macOS / Linux |

## Variables d'environnement

Les scripts reçoivent les variables d'environnement suivantes au moment de l'exécution :

| Variable | Description | Exemple |
| --- | --- | --- |
| `GOPEED_EVENT` | Type d'événement | `DOWNLOAD_DONE` |
| `GOPEED_TASK_ID` | Identifiant unique de la tâche | `abc123-def456` |
| `GOPEED_TASK_NAME` | Nom affiché de la tâche | `ubuntu-24.04.iso` |
| `GOPEED_TASK_STATUS` | Statut de la tâche | `done` |
| `GOPEED_TASK_PATH` | Chemin complet vers le fichier ou dossier téléchargé | `/downloads/ubuntu-24.04.iso` |

`GOPEED_TASK_PATH` s'adapte au type de tâche :
- **Tâches à fichier unique** : Pointe vers le chemin du fichier téléchargé
- **Tâches multi-fichiers** : Pointe vers le chemin du dossier téléchargé

## Exemples de scripts

### Déplacer automatiquement les fichiers vers un répertoire spécifique

```bash
#!/bin/bash
# move_to_hdd.sh - Déplacer automatiquement du SSD vers le HDD après le téléchargement
DEST="/data/hdd/downloads/"
mkdir -p "$DEST"
mv "$GOPEED_TASK_PATH" "$DEST"
echo "Moved $GOPEED_TASK_NAME to $DEST"
```

### Envoyer une notification de fin de téléchargement

```python
#!/usr/bin/env python3
# notify.py - Envoyer une notification après la fin du téléchargement
import os
import urllib.request
import json

task_name = os.environ.get("GOPEED_TASK_NAME", "Unknown")
task_path = os.environ.get("GOPEED_TASK_PATH", "")

# Remplacez par l'URL de votre service de notification
notify_url = "https://your-notify-service.com/api/send"
data = json.dumps({
    "title": "Download Complete",
    "body": f"{task_name}\nPath: {task_path}"
}).encode()

req = urllib.request.Request(notify_url, data=data, headers={"Content-Type": "application/json"})
urllib.request.urlopen(req)
```

## Remarques

- Les scripts ne sont déclenchés que pour les téléchargements **réussis** ; les téléchargements échoués n'exécuteront pas les scripts
- Vous pouvez configurer plusieurs chemins de scripts, ils s'exécuteront dans l'ordre
- Assurez-vous que les fichiers script ont les permissions d'exécution appropriées (utilisez `chmod +x` sur macOS/Linux)
