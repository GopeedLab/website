以下の方法で、すべてのプラットフォームで Gopeed を簡単にダウンロードできます：

1. [公式ウェブサイト](https://gopeed.com) からダウンロードすると、お使いのオペレーティングシステムに応じて自動的に対応バージョンが選択されます。
2. [GitHub](https://github.com/GopeedLab/gopeed/releases) からダウンロードすると、対応するバージョンを自分で選択してダウンロードできます。

## プラットフォーム別インストールガイド

### Windows

Windowsには2つの配布版があります。ポータブル版とインストーラー版です。ポータブル版はインストール不要で、解凍して実行するだけです。インストーラー版はインストールが必要で、インストール手順は他のソフトウェアと同じです。ご自身の好みに合わせてお選びください。

> ポータブル版のファイル名は：Gopeed-v1.x.x-windows-amd64-portable.zip

直接ダウンロードするほか、パッケージマネージャーから Gopeed をインストールまたはアップグレードすることもできます：

- [Scoop](https://github.com/ScoopInstaller/Scoop)：コミュニティが保守する Windows 向けコマンドラインパッケージマネージャーです。軽量で柔軟性があり、管理者権限なしでインストールでき、ソフトウェアの管理と更新が簡単です。

  ```powershell
  scoop bucket add extras
  scoop install extras/gopeed
  ```

- [Winget](https://learn.microsoft.com/en-us/windows/package-manager/winget)：Microsoft公式のパッケージマネージャーで、Windows オペレーティングシステムと深く統合されています。

  ```powershell
  winget install -e --id monkeyWie.Gopeed
  ```

#### ファイアウォールの誤検知

Windows システムでは、一部のアンチウイルスソフトが Gopeed の2つのコアコンポーネントを誤検知する場合があります。ブラウザ拡張機能通信用の `host.exe` と、アプリケーション更新用の `updater.exe` です。ブラウザ拡張機能がダウンロードタスクを正常にインターセプトし、アプリケーションが自動更新できるようにするために、`Gopeed のインストールディレクトリ` をファイアウォールまたはアンチウイルスソフトのホワイトリストに追加してください。

<Callout type="info">
**安全なダウンロード**：このドキュメントで説明されている方法、つまり[公式ウェブサイト](https://gopeed.com)、[公式 GitHub リポジトリ](https://github.com/GopeedLab/gopeed/releases)、または Scoop や Winget から必ず Gopeed をダウンロードしてください。ソフトウェアの安全性を確保するため、サードパーティのソースは使用しないでください。

**誤検知について**：これは Go 言語でコンパイルされたアプリケーションの特性に関連している可能性があり、現時点では良い解決策がありません。将来的にはコンポーネントを Rust に移行してこの問題の改善を検討する予定です。

</Callout>

### Linux

Linux システムでは、`.deb` と `.AppImage` パッケージの両方を提供しており、最適なものをお選びいただけます。また、以下のコマンドで簡単にインストールすることもできます：

- Flatpak

  ```sh
  flatpak install flathub com.gopeed.Gopeed
  ```

- Snap

  ```sh
  sudo snap install gopeed
  ```

### macOS

macOS 版は `.dmg` ファイルを提供しており、ダブルクリックでインストールできます。インストールパッケージは `intel` と `apple silicon` の2つのアーキテクチャに対応しています。

> ヒント：macOS で起動に失敗する場合は、ターミナルで `xattr -d com.apple.quarantine /Applications/Gopeed.app` コマンドを実行してください。

### Android

Android 版は `.apk` ファイルを提供しており、直接ダウンロードしてインストールでき、すべての CPU アーキテクチャに対応しています。

### iOS

現在、iOS プラットフォームでは `.ipa` ファイルのみを提供しており、ご自身で署名してインストールする必要があります。[TrollStore](https://github.com/opa334/TrollStore) を使用したインストールをお勧めします。

> なぜ App Store に掲載されていないのですか？
>
> Apple の審査メカニズムにより、BitTorrent プロトコル関連のアプリは掲載が許可されていません。
>
> なぜ TestFlight に掲載されていないのですか？
>
> お金がないからです！このプロジェクトは純粋に愛と発電で成り立っています。十分なスポンサーシップが得られれば、すぐに TestFlight に掲載します。ですので、ぜひご支援をお願いいたします。オープンソースは容易ではありません。ありがとうございます！

### Web

リモートダウンロードサービスが必要な場合は、Web 版の使用をご検討ください。Gopeed は各プラットフォーム向けの Web 版を提供しています。お使いのシステムと CPU アーキテクチャに合わせてダウンロードしてください。

![web](/images/guide/install/web.png)

以下は Windows プラットフォームでの Web 版の使用例です。他のプラットフォームでも使い方は同様です。

1. Web 版をダウンロードし、解凍するとフォルダが作成されます。任意の場所に配置してください。
2. フォルダのルートディレクトリでターミナルを開き、`./gopeed.exe` を実行します。正常に実行されると、以下のような出力が表示されます：

   ```bash
      _______   ______   .______    _______  _______  _______
    /  _____| /  __  \  |   _  \  |   ____||   ____||       \
    |  |  __  |  |  |  | |  |_)  | |  |__   |  |__   |  .--.  |
    |  | |_ | |  |  |  | |   ___/  |   __|  |   __|  |  |  |  |
    |  |__| | |  `--'  | |  |      |  |____ |  |____ |  '--'  |
    \______|  \______/  | _|      |_______||_______||_______/

    Server start success on http://[::]:9999
   ```

3. ブラウザを開き、`http://localhost:9999` にアクセスします。

#### Web 設定

Web 版はコマンドラインパラメータまたは設定ファイルによる設定に対応しています。コマンドラインパラメータは `./gopeed.exe -h` で確認できます：

```sh
$ ./gopeed.exe -h
Usage of C:\Users\levi\Downloads\gopeed-web-v1.3.13-windows-amd64\gopeed.exe:
  -A string
        Bind Address (default "0.0.0.0")
  -P int
        Bind Port (default 9999)
  -T string
        API token, that can only be used when basic authentication is enabled.
  -c string
        Config file path (default "./config.json")
  -d string
        Storage directory
  -p string
        HTTP Basic Auth Password
  -u string
        HTTP Basic Auth Username (default "gopeed")
```

設定ファイルによる設定にも対応しています。ルートディレクトリに `config.json` ファイルを作成し、以下の内容を記述します：

```json
{
  "address": "",
  "port": 0,
  "username": "",
  "password": "",
  "apiToken": "",
  "storageDir": "",
  "whiteDownloadDirs": ["/root/downloads", "/root/dir/*", "/root/dir?abc"]
}
```

環境変数による設定にも対応しており、ルールは `GOPEED_ConfigKey` です。例えば：

```sh
export GOPEED_ADDRESS="0.0.0.0"
export GOPEED_PORT="9999"
export GOPEED_USERNAME="gopeed"
export GOPEED_PASSWORD="xxx"
export GOPEED_APITOKEN=""
export GOPEED_STORAGEDIR=""
export GOPEED_WHITEDOWNLOADDIRS="/root/downloads,/root/dir/*,/root/dir?abc"
```

<Callout type="warn">
  パブリック IP でデプロイする場合は、必ず認証を有効にしてください。そうしないとセキュリティリスクがあります。
</Callout>

#### デフォルトダウンロード設定

サーバーの初回起動時にデフォルトのダウンロード設定を行う必要がある場合は、`config.json` に `downloadConfig` フィールドを追加できます。設定の詳細は [DownloaderStoreConfig](https://pkg.go.dev/github.com/GopeedLab/gopeed/pkg/base#DownloaderStoreConfig) を参照してください。以下に例を示します：

```json
{
  "address": "127.0.0.1",
  "port": 9999,
  "downloadConfig": {
    "downloadDir": "d:/test"
  }
}
```

### Docker

1行のコマンドで実行できます：

```sh
docker run --name gopeed -d -p 9999:9999 liwei2633/gopeed
```

ダウンロードディレクトリをマウントする場合：

```sh
docker run --name gopeed -d -p 9999:9999 \
    -v /path/to/download:/app/Downloads \
    liwei2633/gopeed
```

データディレクトリをマウントする場合：

```sh
docker run --name gopeed -d -p 9999:9999 \
    -v /path/to/download:/app/Downloads \
    -v /path/to/storage:/app/storage liwei2633/gopeed
```

コンテナのグループ ID とユーザー ID を指定する場合：

```sh
docker run --name gopeed -d -p 9999:9999 \
    -e PGID=100 \
    -e PUID=1000 \
    liwei2633/gopeed
```

認証を有効にする必要がある場合は、環境変数パラメータを渡すことができます：

```sh
docker run --name gopeed -d -p 9999:9999 \
    -e GOPEED_USERNAME="admin" \
    -e GOPEED_PASSWORD="123" \
    -v /path/to/download:/app/Downloads \
    -v /path/to/storage:/app/storage \
    liwei2633/gopeed
```

### コマンドライン

コマンドライン版には `Golang` 環境が必要です。`Golang` 環境がインストールされていない場合は、[こちら](https://golang.org/doc/install)を参照してインストールしてください。

インストールコマンド：

```sh
go install github.com/GopeedLab/gopeed/cmd/gopeed@latest
```

インストール後、`gopeed` コマンドでダウンロードできます。具体的な使い方は `gopeed -h` で確認できます：

```sh
$ gopeed -h
Usage of gopeed:
  -C int
        Concurrent connections. (default 16)
  -D string
        Store directory. (default "C:\\Users\\levi")
```

#### コマンドライン使用例

HTTP リソースをダウンロード：

```sh
gopeed https://example.com/file.zip
```

torrent ファイルをダウンロード：

```sh
gopeed D:/Downloads/file.torrent
```

マグネットリンクをダウンロード：

```sh
gopeed magnet:?xt=urn:btih:xxxx
```

ed2k リンクをダウンロード：

```sh
gopeed 'ed2k://|file|example.iso|123456789|0123456789ABCDEF0123456789ABCDEF|/'
```
