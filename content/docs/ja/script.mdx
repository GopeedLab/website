---
title: スクリプト実行
description: ダウンロード完了後に自動的に実行されるフックスクリプトを設定し、ファイル移動、通知、その他の自動化を実現します。
---

Gopeed は、ダウンロードタスクが **正常に完了** した際に、指定したフックスクリプトを自動的に実行する機能に対応しています。この機能を利用して、ファイルの移動、通知の送信、トランスコードのトリガーなどを自動化できます。

## 有効化方法

1. Gopeed クライアントを開きます
2. **設定** → **詳細** → **開発者** に移動します
3. **スクリプト実行** セクションを見つけ、トグルを有効にします
4. スクリプトファイルのフルパスを追加します

## 対応スクリプトタイプ

Gopeed はファイル拡張子に基づいて自動的にインタプリタを選択します：

| 拡張子 | インタプリタ | プラットフォーム |
| --- | --- | --- |
| `.sh` / `.bash` | Bash | macOS / Linux |
| `.py` | Python 3 | すべてのプラットフォーム |
| `.js` | Node.js | すべてのプラットフォーム |
| `.bat` / `.cmd` | cmd.exe | Windows のみ |
| `.ps1` | PowerShell | すべてのプラットフォーム |
| 拡張子なし | 直接実行（shebang または実行権限が必要） | macOS / Linux |

## 環境変数

スクリプトの実行時には、以下の環境変数が渡されます：

| 変数 | 説明 | 例 |
| --- | --- | --- |
| `GOPEED_EVENT` | イベントタイプ | `DOWNLOAD_DONE` |
| `GOPEED_TASK_ID` | タスクの一意な ID | `abc123-def456` |
| `GOPEED_TASK_NAME` | タスクの表示名 | `ubuntu-24.04.iso` |
| `GOPEED_TASK_STATUS` | タスクのステータス | `done` |
| `GOPEED_TASK_PATH` | ダウンロードされたファイルまたはフォルダーのフルパス | `/downloads/ubuntu-24.04.iso` |

`GOPEED_TASK_PATH` はタスクタイプに応じて適応されます：
- **単一ファイルタスク**：ダウンロードされたファイルのパスを指します
- **複数ファイルタスク**：ダウンロードされたフォルダーのパスを指します

## スクリプト例

### ファイルの自動移動

```bash
#!/bin/bash
# move_to_hdd.sh - ダウンロード後に SSD から HDD へ自動移動
DEST="/data/hdd/downloads/"
mkdir -p "$DEST"
mv "$GOPEED_TASK_PATH" "$DEST"
echo "Moved $GOPEED_TASK_NAME to $DEST"
```

### ダウンロード完了通知の送信

```python
#!/usr/bin/env python3
# notify.py - ダウンロード完了後に通知を送信
import os
import urllib.request
import json

task_name = os.environ.get("GOPEED_TASK_NAME", "Unknown")
task_path = os.environ.get("GOPEED_TASK_PATH", "")

# 通知サービスの URL に置き換えてください
notify_url = "https://your-notify-service.com/api/send"
data = json.dumps({
    "title": "Download Complete",
    "body": f"{task_name}\nPath: {task_path}"
}).encode()

req = urllib.request.Request(notify_url, data=data, headers={"Content-Type": "application/json"})
urllib.request.urlopen(req)
```

## 注意事項

- スクリプトは **正常に完了** したダウンロードでのみトリガーされます。失敗したダウンロードではスクリプトは実行されません
- 複数のスクリプトパスを設定でき、順番に実行されます
- スクリプトファイルに適切な実行権限があることを確認してください（macOS/Linux では `chmod +x` を使用）
