Gopeed は Webhook 設定に対応しており、ダウンロードタスクの完了または失敗時に、指定した URL へ自動的に HTTP POST リクエストを送信します。この機能により、Gotify、Bark、WeCom、DingTalk、Telegram Bot などの通知サービスと連携し、ダウンロード状況をリアルタイムで把握できます。

## 有効化方法

1. Gopeed クライアントを開きます
2. **設定** → **詳細** → **開発者** に移動します
3. **Webhook** セクションを見つけ、クリックして URL を追加します
4. 通知を受け取る HTTP アドレスを入力し、確認をクリックします

## プッシュ形式

Webhook は `Content-Type: application/json` で HTTP POST リクエストを送信します。リクエストボディの構造は以下の通りです：

```json
{
  "event": "DOWNLOAD_DONE",
  "time": 1712000000000,
  "payload": {
    "task": {
      "id": "task-uuid",
      "protocol": "http",
      "meta": {
        "req": {
          "url": "https://example.com/file.zip"
        },
        "opts": {
          "name": "file.zip",
          "path": "/downloads"
        },
        "res": {
          "size": 104857600,
          "files": [
            { "name": "file.zip", "size": 104857600 }
          ]
        }
      }
    }
  }
}
```

### イベントタイプ

| イベント | 説明 |
| --- | --- |
| `DOWNLOAD_DONE` | ダウンロードタスク完了 |
| `DOWNLOAD_ERROR` | ダウンロードタスク失敗 |

## Webhook のテスト

Webhook URL を追加する際、**テスト** ボタンをクリックできます。Gopeed はシミュレートされた `DOWNLOAD_DONE` イベントを指定した URL に送信します。サーバーが HTTP 200 を返した場合のみ、テストは成功と見なされます。

## 複数 URL 対応

複数の Webhook URL を設定できます。ダウンロードイベントは設定されたすべてのアドレスに同時にプッシュされるため、複数の通知チャネルとの連携が容易です。

## 一般的なユースケース

- **プッシュ通知**：Gotify、Bark、Pushover などの自前で構築したプッシュサービスとの連携
- **インスタントメッセージング**：WeCom/DingTalk/Telegram Bot Webhook 経由でダウンロード通知を送信
- **自動化**：解凍、トランスコード、アーカイブなど、後続タスクのトリガー
