Gopeed는 `JavaScript`를 사용한 확장 프로그램 개발을 지원합니다. 확장 프로그램은 Gopeed의 기능을 향상시킬 수 있습니다. 예를 들어 웹사이트에서 비디오나 음악을 다운로드하는 기능 등이 있습니다. [공식 예시](https://github.com/GopeedLab/gopeed-extension-samples/blob/main/README_zh-CN.md)를 통해 빠르게 자세한 내용을 알아볼 수 있습니다.

Gopeed 확장 프로그램은 `git`을 기반으로 탈중앙화된 확장 프로그램 관리를 구현합니다. 확장 프로그램 소스 코드가 원격 git 저장소에 호스팅되어 있으면 Gopeed를 통해 설치하고 업데이트할 수 있습니다. 따라서 `github`, `gitee`, `gitlab` 또는 기타 git 호스팅 플랫폼 모두 확장 프로그램 저장소로 사용할 수 있습니다.

## 빠른 시작

### 스캐폴딩 사용

Gopeed는 확장 프로그램 개발 프로젝트 템플릿을 빠르게 생성할 수 있는 스캐폴딩을 제공합니다:

```sh
npx create-gopeed-ext@latest
```

생성 과정에서 다음과 같은 프롬프트가 표시됩니다:

```sh
√ Project name (gopeed-extension-demo) ...
√ Choose a template » Webpack

Success! Created gopeed-extension-demo at D:\code\study\js\gopeed-extension-demo
Inside that directory, you can run several commands:

  git init
    Initialize git repository

  npm install
    Install dependencies

  npm run dev
    Compiles and hot-reloads for development.

  npm run build
    Compiles and minifies for production.

We suggest that you begin by typing:

  cd gopeed-extension-demo

Happy coding!
```

### 수동 설정

`node.js` 관련 도구에 익숙하지 않은 경우, 수동으로 프로젝트를 생성할 수도 있습니다. 파일 구조는 다음과 같습니다:

```sh
├── index.js
├── manifest.json
```

## 로컬 디버깅

프로젝트가 생성된 후 로컬 디버깅을 진행해야 합니다. 로컬 확장 프로그램 프로젝트를 Gopeed에 설치하여 디버깅할 수 있습니다. 구체적인 단계는 다음과 같습니다:

1. `Gopeed` 개발자 모드를 활성화합니다. 확장 프로그램 페이지에서 설치 버튼을 `연속으로 5번` 클릭하면 개발자 모드가 활성화됩니다.

![](/images/dev/extension/dev-mode.gif)

2. 디렉토리 선택기에서 확장 프로그램 디렉토리를 선택하여 설치합니다.

3. 스캐폴딩에서 `webpack` 모드를 사용하는 경우, `npm run dev`를 통해 자동 컴파일을 시작할 수 있습니다.

4. 작업을 생성하여 확장 프로그램이 적용되는지 확인합니다.

![](/images/dev/extension/demo.gif)

스캐폴딩을 통해 생성된 예시 확장 프로그램은 `https://github.com/hello` 링크를 사용하여 작업을 생성할 때 `example/index.html` 파일을 파싱하는 것을 확인할 수 있습니다.

> 참고: 개발자 모드는 데스크톱 플랫폼에서만 사용할 수 있습니다.

## 개발 설명

이전 섹션에서 기본 확장 프로그램을 만들고 로컬에서 디버깅하는 방법을 알아보았습니다. 하지만 내부적으로는 어떤 일이 일어나고 있을까요?

먼저 `manifest.json` 파일을 살펴보겠습니다. 이 파일은 확장 프로그램의 매니페스트 파일로, 확장 프로그램의 정보를 설명합니다. 각 확장 프로그램 프로젝트는 루트 디렉토리에 `manifest.json` 파일을 포함해야 합니다. 이 섹션의 샘플 파일은 다음과 같습니다:

```json
{
  "name": "gopeed-extention-demo",
  "author": "",
  "title": "gopeed extention demo title",
  "description": "gopeed extention demo description",
  "icon": "",
  "version": "1.0.0",
  "homepage": "",
  "repository": {
    "url": ""
  },
  "scripts": [
    {
      "event": "onResolve",
      "match": {
        "urls": ["*://github.com/*"]
      },
      "entry": "dist/index.js"
    }
  ],
  "settings": []
}
```

각 필드의 의미를 하나씩 소개하겠습니다:

- `name` 및 `author`: Gopeed는 `<author>@<name>`을 확장 프로그램의 ID로 사용합니다. `author`를 입력하면 다른 확장 프로그램과 덮어쓰기 설치되는 것을 방지할 수 있으므로, `author` 필드 입력을 강력히 권장합니다.
- `title` 및 `description`: 확장 프로그램의 제목과 설명입니다.
- `icon`: 확장 프로그램 아이콘으로, 상대 경로를 입력합니다. 예: `icon.png`.
- `version`: 확장 프로그램 버전으로, semver 규격을 사용합니다. 확장 프로그램이 업데이트될 때 이 필드를 기준으로 비교하므로 버전 번호가 규격을 준수하는지 확인하세요.
- `homepage`: 확장 프로그램 홈페이지입니다. 예: `https://gopeed.com`.
- `repository`: 확장 프로그램이 속한 git 저장소 주소입니다. Gopeed 확장 프로그램은 `git`을 통해 탈중앙화된 확장 프로그램 관리를 구현합니다. 따라서 사용자가 확장 프로그램을 설치하고 업데이트하려면 확장 프로그램 소스 코드를 원격 git 저장소에 호스팅하고 이 필드를 구성해야 합니다.

  예시:

  ```json
  {
    "url": "https://github.com/gopeed/gopeed-extension-demo"
  }
  ```

  하나의 git 저장소에 여러 확장 프로그램 프로젝트가 포함된 경우, `directory` 속성을 통해 하위 디렉토리를 지정할 수 있습니다. 예:

  ```json
  {
    "url": "https://github.com/GopeedLab/gopeed-extension-samples",
    "directory": "github-contributor-avatars-sample"
  }
  ```

  Gopeed에서 설치할 때 `#`으로 구분해야 합니다. 예: `https://github.com/GopeedLab/gopeed-extension-samples#github-contributor-avatars-sample`.

- `scripts`: `주의!` Gopeed 확장 프로그램 활성 이벤트 설정입니다.

  샘플 프로젝트에 구성된 `onResolve` 이벤트는 작업을 파싱할 때 트리거됩니다. `match.urls` 필드는 작업 생성 시 URL을 매칭하는 데 사용됩니다. 매칭이 성공하면 `entry` 필드에 지정된 스크립트 파일이 실행됩니다.

  > 매칭 규칙은 Chrome 확장 프로그램의 매칭 규칙과 동일합니다. [여기](https://developer.chrome.com/docs/extensions/mv3/match_patterns/)를 참조하세요

- `settings`: 확장 프로그램 설정입니다. 구성 선언을 통해 Gopeed에서 해당 설정 페이지를 생성하여 사용자 정의 설정을 제공할 수 있습니다. 예를 들어 커스텀 `Cookie`, 커스텀 `User-Agent` 등이 있습니다. 예시:

  ```json
  [
    {
      "name": "cookie",
      "title": "Website Cookie",
      "description": "Cookie can be obtained through browser developer tools",
      "type": "string"
    },
    {
      "name": "quality",
      "title": "Default Quality",
      "type": "number",
      "value": "1080",
      "options": [
        {
          "label": "1080P",
          "value": "1080"
        },
        {
          "label": "720P",
          "value": "720"
        },
        {
          "label": "480P",
          "value": "480"
        }
      ]
    }
  ]
  ```

  - `name`: 설정 항목 이름, 필수.
  - `title`: 설정 항목 제목, 필수.
  - `description`: 설정 항목 설명, 선택사항.
  - `type`: 설정 항목 유형, 가능한 값: `string`, `number`, `boolean`.
  - `value`: 설정 항목 기본값, 선택사항.

## 스크립트 작성

이전 섹션에서 매니페스트를 구성하는 방법을 알아보았습니다. 이제 확장 프로그램 스크립트를 작성하는 방법을 소개하겠습니다.

### 런타임 환경

Gopeed 확장 프로그램 스크립트 엔진은 순수 Go로 작성된 JavaScript 인터프리터인 [goja](https://github.com/dop251/goja)로 구현되었습니다. 하지만 goja는 순수 JS 런타임이므로 `browser` 및 `node.js`의 API는 지원되지 않습니다. 현재 `Gopeed`는 `XMLHttpRequest` 및 `fetch` API를 구현하고 있어, 이 두 API나 이를 기반으로 하는 타사 라이브러리(예: `axios`, `superagent` 등)를 사용하여 네트워크 요청을 구현할 수 있습니다.

또 한 가지 주의할 점은 goja가 대부분의 `es6+` 구문을 기본적으로 지원하지만, `async generator`와 같은 일부 구문은 지원되지 않는다는 것입니다. 하지만 걱정하지 마세요. 스캐폴딩으로 생성된 프로젝트에는 `babel`이 이미 구성되어 있어, 최신 ES 구문을 자유롭게 사용할 수 있으며, 스크립트는 결국 `es5` 구문으로 컴파일됩니다.

### 데모 스크립트 설명

이벤트가 트리거되면 `entry` 필드에 지정된 스크립트 파일이 실행됩니다. 샘플 프로젝트의 스크립트 파일은 다음과 같습니다:

```js
gopeed.events.onResolve((ctx) => {
  ctx.res = {
    name: "example",
    files: [
      {
        name: "index.html",
        req: {
          url: "https://example.com",
        },
      },
    ],
  };
});
```

스크립트 내용을 설명하겠습니다:

- `gopeed.events.onResolve`: `onResolve` 이벤트 핸들러를 등록합니다. 확장 프로그램의 로직이 여기에 포함됩니다.
- `ctx`: 이벤트 컨텍스트로, 현재 이벤트에 대한 정보를 포함합니다. `onResolve` 이벤트에서 `ctx`는 다음을 포함합니다:
  - `req`: 요청 정보로, 리소스 URL, 헤더 등을 포함합니다.
  - `res`: 응답 정보로, 스크립트는 파싱된 파일 목록을 `ctx.res`에 할당해야 하며, Gopeed는 반환된 파일 목록에 따라 다운로드합니다.

요약하자면, `onResolve` 콜백 함수에서 `ctx.req`의 요청 정보를 기반으로 다운로드할 파일 목록을 파싱하고 이를 `ctx.res`에 할당해야 합니다.

> 자세한 확장 프로그램 타이핑 및 관련 소스 코드는 [공식 Gopeed JS 저장소](https://github.com/GopeedLab/gopeed-js)를 참조하세요.
