---

Gopeed는 다운로드 작업이 **성공적으로 완료**되면 지정된 훅 스크립트를 자동으로 실행하는 기능을 지원합니다. 이 기능을 사용하여 파일 이동 자동화, 알림 전송, 트랜스코딩 트리거 등의 작업을 수행할 수 있습니다.

## 활성화 방법

1. Gopeed 클라이언트를 엽니다
2. **설정** → **고급** → **개발자**로 이동합니다
3. **스크립트 실행** 섹션을 찾아 토글을 활성화합니다
4. 스크립트 파일의 전체 경로를 추가합니다

## 지원되는 스크립트 유형

Gopeed는 파일 확장자에 따라 인터프리터를 자동으로 선택합니다:

| 확장자 | 인터프리터 | 플랫폼 |
| --- | --- | --- |
| `.sh` / `.bash` | Bash | macOS / Linux |
| `.py` | Python 3 | 모든 플랫폼 |
| `.js` | Node.js | 모든 플랫폼 |
| `.bat` / `.cmd` | cmd.exe | Windows만 |
| `.ps1` | PowerShell | 모든 플랫폼 |
| 확장자 없음 | 직접 실행 (shebang 또는 실행 권한 필요) | macOS / Linux |

## 환경 변수

스크립트는 실행 시 다음 환경 변수를 수신합니다:

| 변수 | 설명 | 예시 |
| --- | --- | --- |
| `GOPEED_EVENT` | 이벤트 유형 | `DOWNLOAD_DONE` |
| `GOPEED_TASK_ID` | 고유 작업 ID | `abc123-def456` |
| `GOPEED_TASK_NAME` | 작업 표시 이름 | `ubuntu-24.04.iso` |
| `GOPEED_TASK_STATUS` | 작업 상태 | `done` |
| `GOPEED_TASK_PATH` | 다운로드된 파일 또는 폴더의 전체 경로 | `/downloads/ubuntu-24.04.iso` |

`GOPEED_TASK_PATH`는 작업 유형에 따라 조정됩니다:
- **단일 파일 작업**: 다운로드된 파일 경로를 가리킵니다
- **다중 파일 작업**: 다운로드된 폴더 경로를 가리킵니다

## 예시 스크립트

### 파일을 특정 디렉토리로 자동 이동

```bash
#!/bin/bash
# move_to_hdd.sh - 다운로드 후 SSD에서 HDD로 자동 이동
DEST="/data/hdd/downloads/"
mkdir -p "$DEST"
mv "$GOPEED_TASK_PATH" "$DEST"
echo "Moved $GOPEED_TASK_NAME to $DEST"
```

### 다운로드 완료 알림 전송

```python
#!/usr/bin/env python3
# notify.py - 다운로드 완료 후 알림 전송
import os
import urllib.request
import json

task_name = os.environ.get("GOPEED_TASK_NAME", "Unknown")
task_path = os.environ.get("GOPEED_TASK_PATH", "")

# 알림 서비스 URL로 변경하세요
notify_url = "https://your-notify-service.com/api/send"
data = json.dumps({
    "title": "Download Complete",
    "body": f"{task_name}\nPath: {task_path}"
}).encode()

req = urllib.request.Request(notify_url, data=data, headers={"Content-Type": "application/json"})
urllib.request.urlopen(req)
```

## 참고 사항

- 스크립트는 **성공적인** 다운로드에 대해서만 트리거됩니다. 실패한 다운로드는 스크립트를 실행하지 않습니다
- 여러 스크립트 경로를 구성할 수 있으며, 순서대로 실행됩니다
- 스크립트 파일에 적절한 실행 권한이 있는지 확인하세요 (macOS/Linux에서는 `chmod +x` 사용)
