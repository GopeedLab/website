Gopeed는 웹훅 구성을 지원하여, 다운로드 작업이 완료되거나 실패할 때 지정된 URL로 자동으로 HTTP POST 요청을 보냅니다. 이 기능은 Gotify, Bark, WeCom, DingTalk, Telegram Bot 등의 알림 서비스와 연동하여 다운로드 상태를 실시간으로 파악할 수 있게 해줍니다.

## 활성화 방법

1. Gopeed 클라이언트를 엽니다
2. **설정** → **고급** → **개발자**로 이동합니다
3. **Webhook** 섹션을 찾아 URL 추가를 클릭합니다
4. 알림을 수신할 HTTP 주소를 입력하고 확인을 클릭합니다

## 푸시 형식

웹훅은 `Content-Type: application/json`으로 HTTP POST 요청을 보냅니다. 요청 본문 구조는 다음과 같습니다:

```json
{
  "event": "DOWNLOAD_DONE",
  "time": 1712000000000,
  "payload": {
    "task": {
      "id": "task-uuid",
      "protocol": "http",
      "meta": {
        "req": {
          "url": "https://example.com/file.zip"
        },
        "opts": {
          "name": "file.zip",
          "path": "/downloads"
        },
        "res": {
          "size": 104857600,
          "files": [
            { "name": "file.zip", "size": 104857600 }
          ]
        }
      }
    }
  }
}
```

### 이벤트 유형

| 이벤트 | 설명 |
| --- | --- |
| `DOWNLOAD_DONE` | 다운로드 작업 완료 |
| `DOWNLOAD_ERROR` | 다운로드 작업 실패 |

## 웹훅 테스트

Webhook URL을 추가할 때 **테스트** 버튼을 클릭할 수 있습니다. Gopeed가 지정된 URL로 시뮬레이션된 `DOWNLOAD_DONE` 이벤트를 보냅니다. 서버가 HTTP 200을 반환할 때만 테스트가 성공한 것으로 간주됩니다.

## 다중 URL 지원

여러 개의 Webhook URL을 구성할 수 있습니다. 다운로드 이벤트는 구성된 모든 주소로 동시에 푸시되어 여러 알림 채널과 쉽게 연동할 수 있습니다.

## 일반적인 사용 사례

- **푸시 알림**: Gotify, Bark, Pushover와 같은 자체 호스팅 푸시 서비스와 연동
- **인스턴트 메시징**: WeCom/DingTalk/Telegram Bot 웹훅을 통해 다운로드 알림 전송
- **자동화**: 압축 해제, 트랜스코딩, 보관 등 다운스트림 작업 트리거
