O Gopeed suporta o desenvolvimento de extensões usando `JavaScript`. Extensões podem aprimorar as funcionalidades do Gopeed, como baixar vídeos ou músicas de um site. Você pode aprender mais rapidamente através dos [exemplos oficiais](https://github.com/GopeedLab/gopeed-extension-samples/blob/main/README_zh-CN.md).

As extensões do Gopeed são baseadas em `git` para gerenciar extensões de forma descentralizada. Desde que o código-fonte da extensão esteja hospedado em um repositório git remoto, ele pode ser instalado e atualizado através do Gopeed. Portanto, seja `github`, `gitee`, `gitlab` ou outras plataformas de hospedagem git, todas podem ser usadas como repositórios de extensões.

## Início Rápido

### Usando o scaffolding

O Gopeed fornece um scaffolding para ajudá-lo a criar rapidamente um template de projeto de desenvolvimento de extensões:

```sh
npx create-gopeed-ext@latest
```

Durante o processo de criação, você verá os seguintes prompts:

```sh
√ Project name (gopeed-extension-demo) ...
√ Choose a template » Webpack

Success! Created gopeed-extension-demo at D:\code\study\js\gopeed-extension-demo
Inside that directory, you can run several commands:

  git init
    Initialize git repository

  npm install
    Install dependencies

  npm run dev
    Compiles and hot-reloads for development.

  npm run build
    Compiles and minifies for production.

We suggest that you begin by typing:

  cd gopeed-extension-demo

Happy coding!
```

### Configuração manual

Se você não está familiarizado com as ferramentas do ecossistema `node.js`, também pode criar um projeto manualmente. A estrutura de arquivos é a seguinte:

```sh
├── index.js
├── manifest.json
```

## Depuração local

Após construir o projeto, você precisa fazer a depuração local. Você pode instalar o projeto de extensão local no Gopeed para depuração. Os passos específicos são os seguintes:

1. Habilite o modo de desenvolvedor do `Gopeed`. Clique no botão de instalação `5 vezes seguidas` na página de extensões para habilitar o modo de desenvolvedor.

![](/images/dev/extension/dev-mode.gif)

2. Clique no botão para selecionar o diretório da extensão no seletor de diretórios para instalá-la.

3. Se você usar o modo `webpack` no scaffolding, pode iniciar a compilação automática através de `npm run dev`.

4. Crie uma tarefa para ver a extensão em funcionamento.

![](/images/dev/extension/demo.gif)

Pode-se observar que a extensão de exemplo criada através do scaffolding consegue analisar um arquivo `example/index.html` ao criar uma tarefa usando o link `https://github.com/hello`.

> Nota: O modo de desenvolvedor só funciona na plataforma desktop.

## Explicação do desenvolvimento

Na seção anterior, fomos capazes de criar uma extensão básica e depurá-la localmente, mas o que está acontecendo nos bastidores?

Primeiro, vamos dar uma olhada no arquivo `manifest.json`, que é o arquivo de manifesto da extensão. Ele descreve as informações da extensão. Cada projeto de extensão deve conter um arquivo `manifest.json` no diretório raiz. O arquivo de exemplo desta seção é o seguinte:

```json
{
  "name": "gopeed-extention-demo",
  "author": "",
  "title": "gopeed extention demo title",
  "description": "gopeed extention demo description",
  "icon": "",
  "version": "1.0.0",
  "homepage": "",
  "repository": {
    "url": ""
  },
  "scripts": [
    {
      "event": "onResolve",
      "match": {
        "urls": ["*://github.com/*"]
      },
      "entry": "dist/index.js"
    }
  ],
  "settings": []
}
```

A seguir, vamos apresentar o significado de cada campo um por um:

- `name` e `author`: O Gopeed usará `<author>@<name>` como o ID da extensão. Ao preencher o `author`, você garante que a extensão não seja facilmente sobrescrita por outras extensões instaladas, portanto é altamente recomendável preencher o campo `author`.
- `title` e `description`: O título e a descrição da extensão.
- `icon`: Ícone da extensão, preencha o caminho relativo, por exemplo: `icon.png`.
- `version`: Versão da extensão, usando a especificação semver. Quando a extensão é atualizada, a comparação é feita com base neste campo, portanto certifique-se de que o número da versão esteja em conformidade com a especificação.
- `homepage`: Página inicial da extensão, por exemplo: `https://gopeed.com`.
- `repository`: O endereço do repositório git ao qual a extensão pertence. As extensões do Gopeed dependem do `git` para gerenciar extensões de forma descentralizada. Portanto, se sua extensão precisa ser instalada e atualizada pelos usuários, você deve hospedar o código-fonte em um repositório git remoto e configurar este campo.

  Por exemplo：

  ```json
  {
    "url": "https://github.com/gopeed/gopeed-extension-demo"
  }
  ```

  Se um repositório git contém vários projetos de extensão, você pode especificar um subdiretório através do atributo `directory`, por exemplo:

  ```json
  {
    "url": "https://github.com/GopeedLab/gopeed-extension-samples",
    "directory": "github-contributor-avatars-sample"
  }
  ```

  Na instalação pelo Gopeed, você precisa usar `#` para separar, por exemplo: `https://github.com/GopeedLab/gopeed-extension-samples#github-contributor-avatars-sample`.

- `scripts`: `Atenção!` Esta é a configuração do evento de ativação da extensão do Gopeed.

  O evento `onResolve` configurado no projeto de exemplo será acionado ao analisar tarefas. O campo `match.urls` é usado para corresponder à URL criada pela tarefa. Se a correspondência for bem-sucedida, o arquivo de script especificado no campo `entry` será executado.

  > As regras de correspondência são consistentes com as regras de correspondência das extensões do Chrome, que podem ser consultadas [aqui](https://developer.chrome.com/docs/extensions/mv3/match_patterns/)

- `settings`: Configurações da extensão. Através da declaração de configuração, a página de configurações correspondente pode ser gerada no Gopeed para fornecer configurações personalizadas pelo usuário, como `Cookie` personalizado, `User-Agent` personalizado, etc., por exemplo:

  ```json
  [
    {
      "name": "cookie",
      "title": "Website Cookie",
      "description": "Cookie can be obtained through browser developer tools",
      "type": "string"
    },
    {
      "name": "quality",
      "title": "Default Quality",
      "type": "number",
      "value": "1080",
      "options": [
        {
          "label": "1080P",
          "value": "1080"
        },
        {
          "label": "720P",
          "value": "720"
        },
        {
          "label": "480P",
          "value": "480"
        }
      ]
    }
  ]
  ```

  - `name`: Nome do item de configuração, obrigatório.
  - `title`: Título do item de configuração, obrigatório.
  - `description`: Descrição do item de configuração, opcional.
  - `type`: Tipo do item de configuração, valores possíveis: `string`, `number`, `boolean`.
  - `value`: Valor padrão do item de configuração, opcional.

## Escrita de scripts

Na seção anterior, aprendemos como configurar o manifesto. Agora vamos apresentar como escrever o script da extensão.

### Ambiente de execução

O motor de scripts de extensão do Gopeed é implementado por [goja](https://github.com/dop251/goja), que é um interpretador JavaScript escrito em Go puro. No entanto, como o goja é apenas um runtime JS puro, as APIs de `browser` e `node.js` não são suportadas. Atualmente, o `Gopeed` implementa as APIs `XMLHttpRequest` e `fetch`, o que significa que você pode usar essas duas APIs ou bibliotecas de terceiros baseadas nelas para implementar requisições de rede, como `axios`, `superagent`, etc.

Outra coisa a notar é que o goja suporta nativamente a maior parte da sintaxe `es6+`, mas algumas sintaxes não são suportadas, como `async generator`. Mas não se preocupe, o projeto criado pelo scaffolding já foi configurado com `babel`, então você pode usar a sintaxe ES mais recente tranquilamente, e o script eventualmente será compilado para a sintaxe `es5`.

### Explicação do script de demonstração

Quando um evento é acionado, o arquivo de script especificado no campo `entry` será executado. O arquivo de script no projeto de exemplo é o seguinte:

```js
gopeed.events.onResolve((ctx) => {
  ctx.res = {
    name: "example",
    files: [
      {
        name: "index.html",
        req: {
          url: "https://example.com",
        },
      },
    ],
  };
});
```

Vamos explicar o conteúdo do script:

- `gopeed.events.onResolve`: Isso registra o manipulador do evento `onResolve`, onde a lógica da extensão reside.
- `ctx`: O contexto do evento, contendo informações sobre o evento atual. No evento `onResolve`, `ctx` contém:
  - `req`: Informações da requisição, incluindo a URL do recurso, cabeçalhos, etc.
  - `res`: Informações da resposta. O script precisa atribuir a lista de arquivos analisada a `ctx.res`, e o Gopeed fará o download com base na lista de arquivos retornada.

Em resumo, na função de callback `onResolve`, você precisa analisar a lista de arquivos para download com base nas informações de requisição em `ctx.req` e atribuí-la a `ctx.res`.

> Para tipagens detalhadas de extensões e código-fonte relacionado, consulte o [repositório oficial do Gopeed JS](https://github.com/GopeedLab/gopeed-js).
