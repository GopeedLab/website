Os métodos a seguir funcionam em todas as plataformas para baixar o Gopeed rapidamente:

1. Acesse o [site oficial](https://gopeed.com) para baixar, que selecionará automaticamente a versão correspondente ao seu sistema operacional.
2. Acesse o [GitHub](https://github.com/GopeedLab/gopeed/releases) para baixar, onde você pode escolher a versão desejada manualmente.

## Guia de Instalação por Plataforma

### Windows

O Windows possui duas distribuições: uma versão portátil e outra com instalador. A versão portátil não precisa ser instalada, basta descompactar e executar. A versão com instalador precisa ser instalada, e o processo de instalação é o mesmo de outros softwares. Você pode escolher de acordo com suas preferências.

> O nome do arquivo da versão portátil é: Gopeed-v1.x.x-windows-amd64-portable.zip

Além do download direto, você também pode instalar ou atualizar o Gopeed através de gerenciadores de pacotes:

- [Scoop](https://github.com/ScoopInstaller/Scoop): Um gerenciador de pacotes de linha de comando mantido pela comunidade para Windows. É leve e flexível, não requer privilégios de administrador para instalar e facilita o gerenciamento e a atualização de software.

  ```powershell
  scoop bucket add extras
  scoop install extras/gopeed
  ```

- [Winget](https://learn.microsoft.com/pt-br/windows/package-manager/winget): Gerenciador de pacotes oficial da Microsoft, profundamente integrado ao sistema operacional Windows.

  ```powershell
  winget install -e --id monkeyWie.Gopeed
  ```

#### Falso Positivo do Firewall

No sistema Windows, alguns antivírus podem detectar falsamente dois componentes principais do Gopeed: `host.exe` para comunicação com extensões do navegador e `updater.exe` para atualizações do aplicativo. Para garantir que a extensão do navegador possa interceptar corretamente as tarefas de download e que o aplicativo possa atualizar automaticamente, adicione o `diretório de instalação do Gopeed` à lista de permissões do seu firewall ou antivírus.

<Callout type="info">
**Download seguro**: Certifique-se de baixar o Gopeed usando os métodos descritos neste documento — seja pelo [site oficial](https://gopeed.com), pelo [repositório oficial do GitHub](https://github.com/GopeedLab/gopeed/releases) ou através do Scoop ou Winget. Evite usar fontes de terceiros para garantir a segurança do software.

**Sobre falsos positivos**: Isso pode estar relacionado às características de aplicativos compilados com a linguagem Go, e atualmente não há uma boa solução. No futuro, consideraremos migrar os componentes para desenvolvimento em Rust para melhorar essa questão.

</Callout>

### Linux

No sistema Linux, fornecemos pacotes `.deb` e `.AppImage` para instalação, para que você possa escolher o que melhor funciona para você. Além disso, você também pode instalar rapidamente usando os seguintes comandos:

- Flatpak

  ```sh
  flatpak install flathub com.gopeed.Gopeed
  ```

- Snap

  ```sh
  sudo snap install gopeed
  ```

### macOS

A versão macOS fornece um arquivo `.dmg`, que pode ser instalado com um duplo clique. O pacote de instalação suporta duas arquiteturas: `intel` e `apple silicon`.

> Dica: Se o macOS falhar ao abrir, execute o comando `xattr -d com.apple.quarantine /Applications/Gopeed.app` no terminal.

### Android

A versão Android fornece um arquivo `.apk`, que pode ser baixado e instalado diretamente, e suporta todas as arquiteturas de CPU.

### iOS

Atualmente, apenas o arquivo `.ipa` é fornecido para a plataforma iOS, que precisa ser assinado e instalado por conta própria. Recomenda-se usar o [TrollStore](https://github.com/opa334/TrollStore) para instalação.

> Por que não está na App Store?
>
> Devido ao mecanismo de revisão da Apple, aplicativos relacionados ao protocolo BitTorrent não são permitidos na loja.
>
> Por que não está no TestFlight?
>
> Porque não tem dinheiro! O projeto é puramente por amor e geração de conteúdo. Se patrocínio suficiente for obtido, será listado no TestFlight imediatamente. Então, por favor, nos apoie. Código aberto não é fácil. Obrigado!

### Web

Se você precisa de um serviço de download remoto, pode considerar o uso da versão Web. O Gopeed fornece versões Web para várias plataformas. Você pode baixá-las de acordo com seu sistema e arquitetura de CPU.

![web](/images/guide/install/web.png)

Aqui está um exemplo de como usar a versão Web na plataforma Windows. O uso em outras plataformas é semelhante.

1. Baixe a versão Web e, após a descompactação, você obterá uma pasta. Coloque-a onde desejar.
2. Abra o terminal no diretório raiz da pasta e execute `./gopeed.exe`. Se executado com sucesso, você verá a seguinte saída:

   ```bash
      _______   ______   .______    _______  _______  _______
    /  _____| /  __  \  |   _  \  |   ____||   ____||       \
    |  |  __  |  |  |  | |  |_)  | |  |__   |  |__   |  .--.  |
    |  | |_ | |  |  |  | |   ___/  |   __|  |   __|  |  |  |  |
    |  |__| | |  `--'  | |  |      |  |____ |  |____ |  '--'  |
    \______|  \______/  | _|      |_______||_______||_______/

    Server start success on http://[::]:9999
   ```

3. Abra o navegador e acesse `http://localhost:9999`.

#### Configuração Web

A versão Web suporta configuração através de parâmetros de linha de comando ou arquivos de configuração. Os parâmetros de linha de comando podem ser consultados através de `./gopeed.exe -h`:

```sh
$ ./gopeed.exe -h
Usage of C:\Users\levi\Downloads\gopeed-web-v1.3.13-windows-amd64\gopeed.exe:
  -A string
        Bind Address (default "0.0.0.0")
  -P int
        Bind Port (default 9999)
  -T string
        API token, that can only be used when basic authentication is enabled.
  -c string
        Config file path (default "./config.json")
  -d string
        Storage directory
  -p string
        HTTP Basic Auth Password
  -u string
        HTTP Basic Auth Username (default "gopeed")
```

Também suporta configuração através de arquivos de configuração. Crie um arquivo `config.json` no diretório raiz com o seguinte conteúdo:

```json
{
  "address": "",
  "port": 0,
  "username": "",
  "password": "",
  "apiToken": "",
  "storageDir": "",
  "whiteDownloadDirs": ["/root/downloads", "/root/dir/*", "/root/dir?abc"]
}
```

Também suporta configuração através de variáveis de ambiente, com a regra `GOPEED_ConfigKey`, por exemplo:

```sh
export GOPEED_ADDRESS="0.0.0.0"
export GOPEED_PORT="9999"
export GOPEED_USERNAME="gopeed"
export GOPEED_PASSWORD="xxx"
export GOPEED_APITOKEN=""
export GOPEED_STORAGEDIR=""
export GOPEED_WHITEDOWNLOADDIRS="/root/downloads,/root/dir/*,/root/dir?abc"
```

<Callout type="warn">
  Se você estiver implantando em um IP público, certifique-se de habilitar a autenticação
  de identidade, caso contrário haverá riscos de segurança.
</Callout>

#### Configuração Padrão de Download

Se você precisa definir a configuração padrão de download quando o servidor inicia pela primeira vez, pode adicionar o campo `downloadConfig` no `config.json`. Para detalhes da configuração, consulte [DownloaderStoreConfig](https://pkg.go.dev/github.com/GopeedLab/gopeed/pkg/base#DownloaderStoreConfig). Aqui está um exemplo:

```json
{
  "address": "127.0.0.1",
  "port": 9999,
  "downloadConfig": {
    "downloadDir": "d:/test"
  }
}
```

### Docker

Uma linha de comando:

```sh
docker run --name gopeed -d -p 9999:9999 liwei2633/gopeed
```

Montar o diretório de download

```sh
docker run --name gopeed -d -p 9999:9999 \
    -v /path/to/download:/app/Downloads \
    liwei2633/gopeed
```

Montar o diretório de dados

```sh
docker run --name gopeed -d -p 9999:9999 \
    -v /path/to/download:/app/Downloads \
    -v /path/to/storage:/app/storage liwei2633/gopeed
```

Especificar o ID do grupo e do usuário do contêiner

```sh
docker run --name gopeed -d -p 9999:9999 \
    -e PGID=100 \
    -e PUID=1000 \
    liwei2633/gopeed
```

Se você precisa habilitar a autenticação de identidade, pode passar parâmetros de variáveis de ambiente:

```sh
docker run --name gopeed -d -p 9999:9999 \
    -e GOPEED_USERNAME="admin" \
    -e GOPEED_PASSWORD="123" \
    -v /path/to/download:/app/Downloads \
    -v /path/to/storage:/app/storage \
    liwei2633/gopeed
```

### Linha de Comando

A versão de linha de comando requer um ambiente `Golang`. Se você não tem um ambiente `Golang` instalado, pode consultar [aqui](https://golang.org/doc/install) para instalar.

Comando de instalação:

```sh
go install github.com/GopeedLab/gopeed/cmd/gopeed@latest
```

Após a instalação, você pode usar o comando `gopeed` para baixar. Você pode ver o uso específico através de `gopeed -h`:

```sh
$ gopeed -h
Usage of gopeed:
  -C int
        Concurrent connections. (default 16)
  -D string
        Store directory. (default "C:\\Users\\levi")
```

#### Exemplos de Uso da Linha de Comando

Baixar um recurso HTTP:

```sh
gopeed https://example.com/file.zip
```

Baixar um arquivo torrent:

```sh
gopeed D:/Downloads/file.torrent
```

Baixar um link magnet:

```sh
gopeed magnet:?xt=urn:btih:xxxx
```

Baixar um link ed2k:

```sh
gopeed 'ed2k://|file|example.iso|123456789|0123456789ABCDEF0123456789ABCDEF|/'
```
