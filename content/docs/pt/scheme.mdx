O Gopeed registra o protocolo personalizado `gopeed:`, permitindo que você inicie o Gopeed diretamente de um navegador, outro aplicativo ou linha de comando para realizar ações.

## Formato URI

```
gopeed:///create?params=<base64-encoded JSON>
gopeed:///extension?params=<base64-encoded JSON>
```

- `<path>`: Tipo de ação, atualmente suporta `/create` e `/extension`
- `params`: Parâmetros JSON codificados em Base64

## Criar Tarefa de Download

Caminho: `/create`

### Parâmetros

| Campo | Tipo | Descrição |
| --- | --- | --- |
| `req.url` | string | URL de download (HTTP, magnet, ed2k, etc.) |

A estrutura completa dos parâmetros corresponde ao corpo da requisição da API de criação de tarefas. Consulte a [documentação da API de Criação de Tarefas](https://gopeed.com/docs/openapi#tag/tasks/POST/api/v1/tasks) para detalhes.

### Exemplo

JSON de parâmetros:

```json
{
  "req": {
    "url": "https://example.com/file.zip"
  }
}
```

## Instalar Extensão

Caminho: `/extension`

### Parâmetros

| Campo | Tipo | Descrição |
| --- | --- | --- |
| `url` | string | URL do repositório Git da extensão |
| `devMode` | boolean | Se deve instalar em modo de desenvolvimento, padrão é `false` |

### Exemplo

JSON de parâmetros:

```json
{
  "url": "https://github.com/user/gopeed-extension-example",
  "devMode": false
}
```

## Interceptação de Downloads no Android

No Android, o Gopeed registra um Intent Filter padrão de manipulação de downloads. Após a instalação, quando você toca em um link de download em um navegador ou outro aplicativo, o sistema mostrará um seletor de aplicativos onde você pode escolher o Gopeed para lidar com o download.

Cenários suportados:

- Escolher o Gopeed quando um navegador mostra um diálogo de download
- Escolher o Gopeed ao compartilhar links HTTP/HTTPS de outros aplicativos
- Definir o Gopeed como o manipulador de download padrão nas configurações de aplicativos padrão do Android

<Callout type="info">
  Após instalar uma nova versão, o Android pode precisar reescanear os aplicativos para reconhecer novos Intent Filters. Tente limpar o cache do aplicativo, reiniciar o dispositivo ou limpar as preferências de aplicativos padrão nas configurações do sistema.
</Callout>
