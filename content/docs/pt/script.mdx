---
title: Execução de Scripts
description: Configure scripts de hook para executar automaticamente após a conclusão dos downloads, permitindo movimentação de arquivos, notificações e outras automações.
---

O Gopeed suporta a execução automática de scripts de hook especificados quando uma tarefa de download é **concluída com sucesso**. Use este recurso para automatizar a movimentação de arquivos, enviar notificações, acionar transcodificação e muito mais.

## Como Habilitar

1. Abra o cliente Gopeed
2. Vá em **Configurações** → **Avançado** → **Desenvolvedor**
3. Encontre a seção **Execução de Scripts** e ative o interruptor
4. Adicione o caminho completo do seu arquivo de script

## Tipos de Script Suportados

O Gopeed seleciona automaticamente o interpretador com base na extensão do arquivo:

| Extensão | Interpretador | Plataforma |
| --- | --- | --- |
| `.sh` / `.bash` | Bash | macOS / Linux |
| `.py` | Python 3 | Todas as plataformas |
| `.js` | Node.js | Todas as plataformas |
| `.bat` / `.cmd` | cmd.exe | Apenas Windows |
| `.ps1` | PowerShell | Todas as plataformas |
| Sem extensão | Execução direta (requer shebang ou permissão de execução) | macOS / Linux |

## Variáveis de Ambiente

Os scripts recebem as seguintes variáveis de ambiente no momento da execução:

| Variável | Descrição | Exemplo |
| --- | --- | --- |
| `GOPEED_EVENT` | Tipo de evento | `DOWNLOAD_DONE` |
| `GOPEED_TASK_ID` | ID único da tarefa | `abc123-def456` |
| `GOPEED_TASK_NAME` | Nome de exibição da tarefa | `ubuntu-24.04.iso` |
| `GOPEED_TASK_STATUS` | Status da tarefa | `done` |
| `GOPEED_TASK_PATH` | Caminho completo do arquivo ou pasta baixada | `/downloads/ubuntu-24.04.iso` |

`GOPEED_TASK_PATH` se adapta ao tipo de tarefa:
- **Tarefas de arquivo único**: Aponta para o caminho do arquivo baixado
- **Tarefas de múltiplos arquivos**: Aponta para o caminho da pasta baixada

## Exemplos de Scripts

### Mover arquivos automaticamente para um diretório específico

```bash
#!/bin/bash
# move_to_hdd.sh - Mover automaticamente do SSD para o HDD após o download
DEST="/data/hdd/downloads/"
mkdir -p "$DEST"
mv "$GOPEED_TASK_PATH" "$DEST"
echo "Moved $GOPEED_TASK_NAME to $DEST"
```

### Enviar notificação de conclusão de download

```python
#!/usr/bin/env python3
# notify.py - Enviar notificação após a conclusão do download
import os
import urllib.request
import json

task_name = os.environ.get("GOPEED_TASK_NAME", "Unknown")
task_path = os.environ.get("GOPEED_TASK_PATH", "")

# Substitua pela URL do seu serviço de notificação
notify_url = "https://your-notify-service.com/api/send"
data = json.dumps({
    "title": "Download Complete",
    "body": f"{task_name}\nPath: {task_path}"
}).encode()

req = urllib.request.Request(notify_url, data=data, headers={"Content-Type": "application/json"})
urllib.request.urlopen(req)
```

## Observações

- Scripts são acionados apenas em downloads **bem-sucedidos**; downloads falhos não executarão scripts
- Você pode configurar vários caminhos de script, e eles serão executados em ordem
- Certifique-se de que os arquivos de script tenham permissões de execução adequadas (use `chmod +x` no macOS/Linux)
