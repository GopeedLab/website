Gopeed поддерживает разработку расширений на `JavaScript`. Расширения могут расширять функциональность Gopeed, например, загружать видео или музыку с веб-сайтов. Вы можете быстро узнать больше об этом из [официальных примеров](https://github.com/GopeedLab/gopeed-extension-samples/blob/main/README_zh-CN.md).

Расширения Gopeed основаны на `git` для реализации децентрализованного управления расширениями. Пока исходный код расширения размещён в удалённом git-репозитории, оно может быть установлено и обновлено через Gopeed. Поэтому будь то `github`, `gitee`, `gitlab` или другие платформы хостинга git — все они могут использоваться в качестве репозиториев расширений.

## Быстрый старт

### Использование скаффолдинга

Gopeed предоставляет скаффолдинг для быстрого создания шаблона проекта разработки расширения:

```sh
npx create-gopeed-ext@latest
```

В процессе создания вы увидите следующие подсказки:

```sh
√ Project name (gopeed-extension-demo) ...
√ Choose a template » Webpack

Success! Created gopeed-extension-demo at D:\code\study\js\gopeed-extension-demo
Inside that directory, you can run several commands:

  git init
    Initialize git repository

  npm install
    Install dependencies

  npm run dev
    Compiles and hot-reloads for development.

  npm run build
    Compiles and minifies for production.

We suggest that you begin by typing:

  cd gopeed-extension-demo

Happy coding!
```

### Ручная настройка

Если вы не знакомы с инструментами `node.js`, вы также можете вручную создать проект со следующей структурой файлов:

```sh
├── index.js
├── manifest.json
```

## Локальная отладка

После создания проекта необходимо выполнить локальную отладку. Вы можете установить локальный проект расширения в Gopeed для отладки. Конкретные шаги:

1. Включите режим разработчика `Gopeed` — нажмите кнопку установки `5 раз подряд` на странице расширений, чтобы включить режим разработчика.

![](/images/dev/extension/dev-mode.gif)

2. Нажмите кнопку и выберите директорию расширения в диалоге выбора для установки.

3. Если вы используете режим `webpack` в скаффолдинге, вы можете запустить автоматическую компиляцию через `npm run dev`.

4. Создайте задачу, чтобы увидеть, как расширение работает.

![](/images/dev/extension/demo.gif)

Видно, что пример расширения, созданный через скаффолдинг, может анализировать файл `example/index.html` при создании задачи с использованием ссылки `https://github.com/hello`.

> Примечание: Режим разработчика доступен только на десктопной платформе.

## Описание разработки

В предыдущем разделе мы смогли создать базовое расширение и отладить его локально, но что происходит под капотом?

Сначала давайте рассмотрим файл `manifest.json` — это манифест расширения, описывающий информацию о расширении. Каждый проект расширения должен содержать файл `manifest.json` в корневой директории. Пример файла в этом разделе:

```json
{
  "name": "gopeed-extention-demo",
  "author": "",
  "title": "gopeed extention demo title",
  "description": "gopeed extention demo description",
  "icon": "",
  "version": "1.0.0",
  "homepage": "",
  "repository": {
    "url": ""
  },
  "scripts": [
    {
      "event": "onResolve",
      "match": {
        "urls": ["*://github.com/*"]
      },
      "entry": "dist/index.js"
    }
  ],
  "settings": []
}
```

Теперь рассмотрим значение каждого поля:

- `name` и `author`: Gopeed использует `<author>@<name>` в качестве идентификатора расширения. Заполнив `author`, вы гарантируете, что расширение не будет случайно перезаписано другими расширениями, поэтому настоятельно рекомендуется заполнять поле `author`.
- `title` и `description`: Название и описание расширения.
- `icon`: Иконка расширения, укажите относительный путь, например: `icon.png`.
- `version`: Версия расширения, используется спецификация semver. При обновлении расширения сравнение происходит по этому полю, поэтому убедитесь, что номер версии соответствует спецификации.
- `homepage`: Домашняя страница расширения, например: `https://gopeed.com`.
- `repository`: Адрес git-репозитория, к которому принадлежит расширение. Расширения Gopeed используют `git` для децентрализованного управления расширениями. Поэтому, если вы хотите, чтобы пользователи могли устанавливать и обновлять ваше расширение, вы должны разместить исходный код в удалённом git-репозитории и настроить это поле.

  Например:

  ```json
  {
    "url": "https://github.com/gopeed/gopeed-extension-demo"
  }
  ```

  Если git-репозиторий содержит несколько проектов расширений, вы можете указать подкаталог через атрибут `directory`, например:

  ```json
  {
    "url": "https://github.com/GopeedLab/gopeed-extension-samples",
    "directory": "github-contributor-avatars-sample"
  }
  ```

  При установке в Gopeed необходимо использовать `#` в качестве разделителя, например: `https://github.com/GopeedLab/gopeed-extension-samples#github-contributor-avatars-sample`.

- `scripts`: `Обратите внимание!` Это конфигурация событий активации расширения Gopeed.

  Событие `onResolve`, настроенное в примере проекта, срабатывает при анализе задач. Поле `match.urls` используется для сопоставления URL-адреса, созданного задачей. При успешном совпадении будет выполнен файл скрипта, указанный в поле `entry`.

  > Правила сопоставления совпадают с правилами сопоставления расширений Chrome, подробнее см. [здесь](https://developer.chrome.com/docs/extensions/mv3/match_patterns/)

- `settings`: Настройки расширения. Через объявление конфигурации в Gopeed может быть сгенерирована соответствующая страница настроек для пользовательских параметров, таких как пользовательский `Cookie`, пользовательский `User-Agent` и т. д., например:

  ```json
  [
    {
      "name": "cookie",
      "title": "Website Cookie",
      "description": "Cookie can be obtained through browser developer tools",
      "type": "string"
    },
    {
      "name": "quality",
      "title": "Default Quality",
      "type": "number",
      "value": "1080",
      "options": [
        {
          "label": "1080P",
          "value": "1080"
        },
        {
          "label": "720P",
          "value": "720"
        },
        {
          "label": "480P",
          "value": "480"
        }
      ]
    }
  ]
  ```

  - `name`: Имя параметра, обязательно.
  - `title`: Заголовок параметра, обязательно.
  - `description`: Описание параметра, необязательно.
  - `type`: Тип параметра, допустимые значения: `string`, `number`, `boolean`.
  - `value`: Значение параметра по умолчанию, необязательно.

## Написание скриптов

В предыдущем разделе мы узнали, как настроить манифест, а теперь рассмотрим, как писать скрипты расширения.

### Среда выполнения

Скриптовый движок расширений Gopeed реализован на базе [goja](https://github.com/dop251/goja) — интерпретатора JavaScript, написанного на чистом Go. Однако, поскольку goja является только средой выполнения JS, API `browser` и `node.js` не поддерживаются. В настоящее время `Gopeed` реализует API `XMLHttpRequest` и `fetch`, что означает, что вы можете использовать эти два API или сторонние библиотеки на их основе для выполнения сетевых запросов, такие как `axios`, `superagent` и т. д.

Также следует отметить, что goja изначально поддерживает большую часть синтаксиса `es6+`, но некоторые конструкции не поддерживаются, например, `async generator`. Но это не проблема — проект, созданный через скаффолдинг, уже настроен с `babel`, поэтому вы можете свободно использовать новейший синтаксис ES, и скрипт в итоге будет скомпилирован в синтаксис `es5`.

### Описание демо-скрипта

Когда событие срабатывает, выполняется файл скрипта, указанный в поле `entry`. Скрипт из примера проекта выглядит следующим образом:

```js
gopeed.events.onResolve((ctx) => {
  ctx.res = {
    name: "example",
    files: [
      {
        name: "index.html",
        req: {
          url: "https://example.com",
        },
      },
    ],
  };
});
```

Рассмотрим содержимое скрипта:

- `gopeed.events.onResolve`: Регистрирует обработчик события `onResolve`, в котором resides логика расширения.
- `ctx`: Контекст события, содержащий информацию о текущем событии. В событии `onResolve` `ctx` содержит:
  - `req`: Информация о запросе, включая URL-адрес ресурса, заголовки и т. д.
  - `res`: Информация об ответе, скрипт должен присвоить список проанализированных файлов в `ctx.res`, и Gopeed выполнит загрузку в соответствии с возвращённым списком файлов.

Проще говоря, в функции обратного вызова `onResolve` необходимо проанализировать список файлов для загрузки на основе информации запроса из `ctx.req` и присвоить его в `ctx.res`.

> Подробнее о типах расширений и связанном исходном коде см. в [официальном репозитории Gopeed JS](https://github.com/GopeedLab/gopeed-js).

## Хранилище расширения

Gopeed предоставляет API для хранения, чтобы поддерживать постоянные данные расширения, например `токены входа`. Пример:

```js
gopeed.events.onResolve((ctx) => {
  // Получаем token; если его нет, выполняем вход
  const token = gopeed.storage.get("token");
  if (!token) {
    const token = await login();
    gopeed.storage.set("token", token);
  }

  // Затем делаем что-то с token
  // ...
});
```

> Примечание: Подробную информацию об API см. в [документации](https://docs.gopeed.com/site/reference/interfaces/gopeed.Storage.html).

## Отладка расширения

В скриптах можно использовать объект `gopeed.logger` для вывода логов. Он поддерживает уровни `debug`, `info`, `warn` и `error`. Пример:

```js
gopeed.logger.debug("debug");
gopeed.logger.info("info");
gopeed.logger.warn("warn");
gopeed.logger.error("error");
```

Файлы логов находятся в каталоге `logs` внутри каталога установки Gopeed, имя файла — `extension.log`. Для просмотра логов в реальном времени можно использовать `tail -f extension.log`.

> Примечание: Логи уровня `debug` работают только для расширений, установленных в режиме разработчика.

## Публикация расширения

После завершения разработки, если проект был создан через скаффолдинг и использует `webpack`, его нужно собрать:

```sh
npm run build
```

Затем нужно создать `удалённый репозиторий`. Например, если на `github` создан репозиторий `https://github.com/xxx/gopeed-extension-demo`, соответствующим образом обновите поле `repository` в `manifest.json`:

```json
{
  "repository": {
    "url": "https://github.com/xxx/gopeed-extension-demo"
  }
}
```

> Правильная настройка `repository` позволяет расширению получать удалённые обновления. Если расширение находится в подкаталоге внутри git-репозитория, можно указать подкаталог через атрибут `directory`, например:
>
> ```json
> {
>   "repository": {
>     "url": "https://github.com/xxx/gopeed-extension-demo",
>     "directory": "path"
>   }
> }
> ```

Также не забудьте корректно задать поля `author` и `name`, чтобы снизить риск совпадения имени с другими расширениями.

Затем отправьте проект в удалённый репозиторий, чтобы завершить публикацию. Чтобы пользователям было проще находить расширения Gopeed на `github`, рекомендуется использовать префикс `gopeed-extension-` в названии проекта, например `gopeed-extension-demo`, и помечать проект тегом `gopeed-extension` на `github`.

## Установка расширения

После публикации в удалённый репозиторий его можно установить в Gopeed. Откройте страницу расширений, введите HTTP-адрес `git clone` расширения (можно опустить суффикс `.git`) и нажмите кнопку `Установить`.

> Примечание: Если расширение находится в подкаталоге, добавьте к адресу `#`, а затем имя подкаталога, например `https://github.com/xxx/gopeed-extension-demo#path`.

## Официальные примеры

Gopeed предоставляет два показательных примера расширений для справки:

- [github-contributor-avatars-sample](https://github.com/GopeedLab/gopeed-extension-samples/tree/main/github-contributor-avatars-sample)

  Это расширение зависит от `node.js` и подходит для сложных сценариев. Оно использует библиотеку `cheerio` для разбора `DOM` страницы и получения списка файлов для загрузки.

- [github-release-sample](https://github.com/GopeedLab/gopeed-extension-samples/tree/main/github-release-sample)

  Это расширение представляет собой проект на `чистом JS` без зависимостей. Оно подходит для простых сценариев и использует `fetch` для сетевых запросов и получения списка файлов для загрузки.
