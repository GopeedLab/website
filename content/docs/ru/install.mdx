Следующие способы подходят для быстрой загрузки Gopeed на всех платформах:

1. Перейдите на [официальный сайт](https://gopeed.com) для загрузки — версия будет автоматически выбрана в соответствии с вашей операционной системой.
2. Перейдите на [GitHub](https://github.com/GopeedLab/gopeed/releases) для загрузки — вы можете самостоятельно выбрать нужную версию.

## Руководство по установке для конкретных платформ

### Windows

Для Windows доступно два дистрибутива: портативная версия и версия с установщиком. Портативная версия не требует установки — достаточно распаковать архив и запустить. Версия с установщиком требует установки, процесс такой же, как и для другого программного обеспечения. Вы можете выбрать подходящий вариант.

> Имя файла портативной версии: Gopeed-v1.x.x-windows-amd64-portable.zip

Помимо прямой загрузки, вы также можете установить или обновить Gopeed через пакетные менеджеры:

- [Scoop](https://github.com/ScoopInstaller/Scoop): Менеджер пакетов командной строки для Windows, поддерживаемый сообществом. Лёгкий и гибкий, не требует прав администратора для установки, упрощает управление и обновление программного обеспечения.

  ```powershell
  scoop bucket add extras
  scoop install extras/gopeed
  ```

- [Winget](https://learn.microsoft.com/ru-ru/windows/package-manager/winget): Официальный пакетный менеджер от Microsoft, глубоко интегрированный в операционную систему Windows.

  ```powershell
  winget install -e --id monkeyWie.Gopeed
  ```

#### Ложное срабатывание брандмауэра

В системе Windows некоторые антивирусные программы могут ложно обнаруживать два основных компонента Gopeed: `host.exe` для связи с расширениями браузера и `updater.exe` для обновления приложения. Чтобы обеспечить корректную работу расширения браузера по перехвату задач загрузки и автоматическое обновление приложения, добавьте `директорию установки Gopeed` в белый список вашего брандмауэра или антивирусного ПО.

<Callout type="info">
**Безопасная загрузка**: Обязательно загружайте Gopeed способами, описанными в этом документе — с [официального сайта](https://gopeed.com), из [официального репозитория GitHub](https://github.com/GopeedLab/gopeed/releases) или через Scoop/Winget. Избегайте использования сторонних источников для обеспечения безопасности программного обеспечения.

**О ложных срабатываниях**: Это может быть связано с особенностями приложений, скомпилированных на языке Go, и в настоящее время нет хорошего решения. В будущем мы рассмотрим возможность миграции компонентов на разработку на Rust для решения этой проблемы.

</Callout>

### Linux

В системе Linux мы предоставляем пакеты `.deb` и `.AppImage` для установки — вы можете выбрать подходящий вариант. Кроме того, вы можете быстро установить с помощью следующих команд:

- Flatpak

  ```sh
  flatpak install flathub com.gopeed.Gopeed
  ```

- Snap

  ```sh
  sudo snap install gopeed
  ```

### macOS

Версия для macOS предоставляет файл `.dmg`, который можно установить двойным щелчком. Установочный пакет поддерживает две архитектуры: `intel` и `apple silicon`.

> Совет: Если приложение не открывается в macOS, выполните команду `xattr -d com.apple.quarantine /Applications/Gopeed.app` в терминале

### Android

Версия для Android предоставляет файл `.apk`, который можно загрузить и установить напрямую, поддерживает все архитектуры процессоров.

### iOS

В настоящее время для платформы iOS предоставляется только файл `.ipa`, который необходимо подписать и установить самостоятельно. Рекомендуется использовать [TrollStore](https://github.com/opa334/TrollStore) для установки.

> Почему нет в App Store?
>
> Из-за механизма проверки Apple, приложения, связанные с протоколом BitTorrent, не допускаются к публикации.
>
> Почему нет в TestFlight?
>
> Потому что нет денег! Проект существует исключительно на энтузиазме. Если будет получено достаточно спонсорской помощи, он будет немедленно размещён в TestFlight. Поэтому, пожалуйста, поддержите нас. Открытый исходный код — это непросто. Спасибо!

### Web

Если вам нужна служба удалённых загрузок, вы можете использовать веб-версию. Gopeed предоставляет веб-версии для различных платформ. Вы можете загрузить нужную версию в зависимости от вашей системы и архитектуры процессора.

![web](/images/guide/install/web.png)

Ниже приведён пример использования веб-версии на платформе Windows, на других платформах использование аналогично.

1. Загрузите веб-версию, после распаковки вы получите папку. Поместите её в удобное для вас место.
2. Откройте терминал в корневой директории папки и выполните `./gopeed.exe`. При успешном запуске вы увидите следующий вывод:

   ```bash
      _______   ______   .______    _______  _______  _______
    /  _____| /  __  \  |   _  \  |   ____||   ____||       \
    |  |  __  |  |  |  | |  |_)  | |  |__   |  |__   |  .--.  |
    |  | |_ | |  |  |  | |   ___/  |   __|  |   __|  |  |  |  |
    |  |__| | |  `--'  | |  |      |  |____ |  |____ |  '--'  |
    \______|  \______/  | _|      |_______||_______||_______/

    Server start success on http://[::]:9999
   ```

3. Откройте браузер и перейдите по адресу `http://localhost:9999`.

#### Конфигурация веб-версии

Веб-версия поддерживает конфигурацию через параметры командной строки или файл конфигурации. Параметры командной строки можно просмотреть через `./gopeed.exe -h`:

```sh
$ ./gopeed.exe -h
Usage of C:\Users\levi\Downloads\gopeed-web-v1.3.13-windows-amd64\gopeed.exe:
  -A string
        Bind Address (default "0.0.0.0")
  -P int
        Bind Port (default 9999)
  -T string
        API token, that can only be used when basic authentication is enabled.
  -c string
        Config file path (default "./config.json")
  -d string
        Storage directory
  -p string
        HTTP Basic Auth Password
  -u string
        HTTP Basic Auth Username (default "gopeed")
```

Также поддерживается конфигурация через файл конфигурации. Создайте файл `config.json` в корневой директории со следующим содержимым:

```json
{
  "address": "",
  "port": 0,
  "username": "",
  "password": "",
  "apiToken": "",
  "storageDir": "",
  "whiteDownloadDirs": ["/root/downloads", "/root/dir/*", "/root/dir?abc"]
}
```

Также поддерживается конфигурация через переменные окружения, по правилу `GOPEED_ConfigKey`, например:

```sh
export GOPEED_ADDRESS="0.0.0.0"
export GOPEED_PORT="9999"
export GOPEED_USERNAME="gopeed"
export GOPEED_PASSWORD="xxx"
export GOPEED_APITOKEN=""
export GOPEED_STORAGEDIR=""
export GOPEED_WHITEDOWNLOADDIRS="/root/downloads,/root/dir/*,/root/dir?abc"
```

<Callout type="warn">
  Если вы развёртываете на публичном IP-адресе, обязательно включите аутентификацию,
  иначе возникнут риски безопасности.
</Callout>

#### Конфигурация загрузки по умолчанию

Если вам нужно настроить конфигурацию загрузки по умолчанию при первом запуске сервера, вы можете добавить поле `downloadConfig` в `config.json`. Подробнее о конфигурации см. в [DownloaderStoreConfig](https://pkg.go.dev/github.com/GopeedLab/gopeed/pkg/base#DownloaderStoreConfig). Пример:

```json
{
  "address": "127.0.0.1",
  "port": 9999,
  "downloadConfig": {
    "downloadDir": "d:/test"
  }
}
```

### Docker

Одна команда:

```sh
docker run --name gopeed -d -p 9999:9999 liwei2633/gopeed
```

Монтирование директории загрузок

```sh
docker run --name gopeed -d -p 9999:9999 \
    -v /path/to/download:/app/Downloads \
    liwei2633/gopeed
```

Монтирование директории данных

```sh
docker run --name gopeed -d -p 9999:9999 \
    -v /path/to/download:/app/Downloads \
    -v /path/to/storage:/app/storage liwei2633/gopeed
```

Указание ID группы и пользователя контейнера

```sh
docker run --name gopeed -d -p 9999:9999 \
    -e PGID=100 \
    -e PUID=1000 \
    liwei2633/gopeed
```

Если вам нужно включить аутентификацию, можно передать переменные окружения:

```sh
docker run --name gopeed -d -p 9999:9999 \
    -e GOPEED_USERNAME="admin" \
    -e GOPEED_PASSWORD="123" \
    -v /path/to/download:/app/Downloads \
    -v /path/to/storage:/app/storage \
    liwei2633/gopeed
```

### Командная строка

Для версии командной строки требуется среда `Golang`. Если у вас не установлена среда `Golang`, обратитесь к [инструкции по установке](https://golang.org/doc/install).

Команда установки:

```sh
go install github.com/GopeedLab/gopeed/cmd/gopeed@latest
```

После установки вы можете использовать команду `gopeed` для загрузок. Подробнее об использовании можно узнать через `gopeed -h`:

```sh
$ gopeed -h
Usage of gopeed:
  -C int
        Concurrent connections. (default 16)
  -D string
        Store directory. (default "C:\\Users\\levi")
```

#### Примеры использования командной строки

Загрузка HTTP-ресурса:

```sh
gopeed https://example.com/file.zip
```

Загрузка торрент-файла:

```sh
gopeed D:/Downloads/file.torrent
```

Загрузка magnet-ссылки:

```sh
gopeed magnet:?xt=urn:btih:xxxx
```

Загрузка ed2k-ссылки:

```sh
gopeed 'ed2k://|file|example.iso|123456789|0123456789ABCDEF0123456789ABCDEF|/'
```
