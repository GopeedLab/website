Gopeed поддерживает автоматическое выполнение указанных скриптов-перехватчиков при **успешном завершении** задачи загрузки. Используйте эту функцию для автоматического перемещения файлов, отправки уведомлений, запуска транскодирования и многого другого.

## Как включить

1. Откройте клиент Gopeed
2. Перейдите в **Настройки** → **Дополнительно** → **Разработчик**
3. Найдите раздел **Выполнение скриптов** и включите переключатель
4. Добавьте полный путь к файлу скрипта

## Поддерживаемые типы скриптов

Gopeed автоматически выбирает интерпретатор на основе расширения файла:

| Расширение | Интерпретатор | Платформа |
| --- | --- | --- |
| `.sh` / `.bash` | Bash | macOS / Linux |
| `.py` | Python 3 | Все платформы |
| `.js` | Node.js | Все платформы |
| `.bat` / `.cmd` | cmd.exe | Только Windows |
| `.ps1` | PowerShell | Все платформы |
| Без расширения | Прямое выполнение (требуется shebang или права на выполнение) | macOS / Linux |

## Переменные окружения

Скрипты получают следующие переменные окружения при выполнении:

| Переменная | Описание | Пример |
| --- | --- | --- |
| `GOPEED_EVENT` | Тип события | `DOWNLOAD_DONE` |
| `GOPEED_TASK_ID` | Уникальный идентификатор задачи | `abc123-def456` |
| `GOPEED_TASK_NAME` | Отображаемое имя задачи | `ubuntu-24.04.iso` |
| `GOPEED_TASK_STATUS` | Статус задачи | `done` |
| `GOPEED_TASK_PATH` | Полный путь к загруженному файлу или папке | `/downloads/ubuntu-24.04.iso` |

`GOPEED_TASK_PATH` адаптируется к типу задачи:
- **Задачи с одним файлом**: Указывает путь к загруженному файлу
- **Задачи с несколькими файлами**: Указывает путь к загруженной папке

## Примеры скриптов

### Автоматическое перемещение файлов в определённую директорию

```bash
#!/bin/bash
# move_to_hdd.sh - Автоматическое перемещение с SSD на HDD после загрузки
DEST="/data/hdd/downloads/"
mkdir -p "$DEST"
mv "$GOPEED_TASK_PATH" "$DEST"
echo "Moved $GOPEED_TASK_NAME to $DEST"
```

### Отправка уведомления о завершении загрузки

```python
#!/usr/bin/env python3
# notify.py - Отправка уведомления после завершения загрузки
import os
import urllib.request
import json

task_name = os.environ.get("GOPEED_TASK_NAME", "Unknown")
task_path = os.environ.get("GOPEED_TASK_PATH", "")

# Замените на URL вашего сервиса уведомлений
notify_url = "https://your-notify-service.com/api/send"
data = json.dumps({
    "title": "Download Complete",
    "body": f"{task_name}\nPath: {task_path}"
}).encode()

req = urllib.request.Request(notify_url, data=data, headers={"Content-Type": "application/json"})
urllib.request.urlopen(req)
```

## Примечания

- Скрипты запускаются только при **успешных** загрузках; при неудачных загрузках скрипты не выполняются
- Можно настроить несколько путей к скриптам, и они будут выполняться по порядку
- Убедитесь, что файлы скриптов имеют соответствующие права на выполнение (используйте `chmod +x` на macOS/Linux)
