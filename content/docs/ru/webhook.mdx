Gopeed поддерживает настройку вебхуков, автоматически отправляя HTTP POST запросы на указанный URL при завершении или ошибке задачи загрузки. Эта функция интегрируется с такими сервисами уведомлений, как Gotify, Bark, WeCom, DingTalk и Telegram Bot, чтобы вы были в курсе статуса загрузок в реальном времени.

## Как включить

1. Откройте клиент Gopeed
2. Перейдите в **Настройки** → **Дополнительно** → **Разработчик**
3. Найдите раздел **Webhook** и нажмите, чтобы добавить URL
4. Введите HTTP-адрес для получения уведомлений и нажмите подтвердить

## Формат отправки

Вебхуки используют HTTP POST запросы с `Content-Type: application/json`. Структура тела запроса:

```json
{
  "event": "DOWNLOAD_DONE",
  "time": 1712000000000,
  "payload": {
    "task": {
      "id": "task-uuid",
      "protocol": "http",
      "meta": {
        "req": {
          "url": "https://example.com/file.zip"
        },
        "opts": {
          "name": "file.zip",
          "path": "/downloads"
        },
        "res": {
          "size": 104857600,
          "files": [
            { "name": "file.zip", "size": 104857600 }
          ]
        }
      }
    }
  }
}
```

### Типы событий

| Событие | Описание |
| --- | --- |
| `DOWNLOAD_DONE` | Задача загрузки завершена |
| `DOWNLOAD_ERROR` | Ошибка задачи загрузки |

## Тестирование вебхука

При добавлении URL вебхука вы можете нажать кнопку **Тест**. Gopeed отправит смоделированное событие `DOWNLOAD_DONE` на указанный URL. Тест считается успешным только тогда, когда сервер возвращает HTTP 200.

## Поддержка нескольких URL

Вы можете настроить несколько URL вебхуков. События загрузки будут отправлены на все настроенные адреса одновременно, что упрощает интеграцию с несколькими каналами уведомлений.

## Типичные сценарии использования

- **Push-уведомления**: Интеграция с собственными сервисами push-уведомлений, такими как Gotify, Bark или Pushover
- **Мессенджеры**: Отправка уведомлений о загрузках через вебхуки WeCom/DingTalk/Telegram Bot
- **Автоматизация**: Запуск последующих задач, таких как распаковка, транскодирование или архивирование
