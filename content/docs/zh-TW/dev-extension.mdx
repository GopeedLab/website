Gopeed 支援使用 `JavaScript` 進行擴充開發，擴充功能可以增強 Gopeed 的功能，例如下載某個網站的影片，或者下載某個網站的音樂等等，可以透過[官方範例](https://github.com/GopeedLab/gopeed-extension-samples/blob/main/README_zh-CN.md)快速了解一下。

Gopeed 擴充功能基於 `git` 來實現去中心化的擴充管理，只要將擴充原始碼託管到遠端 git 存放庫中，就可以透過 Gopeed 進行安裝和更新，所以無論是 `github`、`gitee`、`gitlab` 還是其它 git 託管平台，都可以作為擴充的託管存放庫。

## 快速開始

### 使用腳手架搭建

Gopeed 提供了腳手架來幫助你快速建立一個擴充開發專案範本：

```sh
npx create-gopeed-ext@latest
```

在建立過程中，將會看到以下提示：

```sh
√ Project name (gopeed-extension-demo) ...
√ Choose a template » Webpack

Success! Created gopeed-extension-demo at D:\code\study\js\gopeed-extension-demo
Inside that directory, you can run several commands:

  git init
    Initialize git repository

  npm install
    Install dependencies

  npm run dev
    Compiles and hot-reloads for development.

  npm run build
    Compiles and minifies for production.

We suggest that you begin by typing:

  cd gopeed-extension-demo

Happy coding!
```

### 手動搭建

如果你對 `node.js` 周邊工具不太熟悉，也可以手動建立一個專案，檔案結構如下：

```sh
├── index.js
├── manifest.json
```

## 本機偵錯

在專案搭建好之後，需要進行本機偵錯，可以將本機的擴充專案安裝到 Gopeed 中進行除錯，具體步驟如下：

1. 開啟 `Gopeed` 開發者模式，在擴充頁面 `連續點擊 5 次` 安裝按鈕，即可開啟開發者模式。

![](/images/dev/extension/dev-mode.gif)

2. 點擊按鈕在目錄選擇器中選擇擴充目錄，進行安裝。

3. 如果使用腳手架中的 `webpack` 模式，可以透過 `npm run dev` 啟動自動編譯。

4. 建立任務即可看到擴充功能生效了。

![](/images/dev/extension/demo.gif)

可以看到透過腳手架建立的範例擴充功能，可以實現使用 `https://github.com/hello` 連結建立任務時，解析出一個 `example/index.html` 檔案

> 註：開發者模式只在桌面平台有效。

## 開發解析

上一節中，已經能夠建立一個基礎的擴充功能並進行本機偵錯了，但是在面紗之下，它究竟是怎麼運作的呢？

首先我們來看一下 `manifest.json` 檔案，這個是擴充功能的清單檔案，它描述了擴充的資訊，每個擴充專案根目錄中必須包含一個 `manifest.json` 檔案，本節中的範例檔案如下：

```json
{
  "name": "gopeed-extention-demo",
  "author": "",
  "title": "gopeed extention demo title",
  "description": "gopeed extention demo description",
  "icon": "",
  "version": "1.0.0",
  "homepage": "",
  "repository": {
    "url": ""
  },
  "scripts": [
    {
      "event": "onResolve",
      "match": {
        "urls": ["*://github.com/*"]
      },
      "entry": "dist/index.js"
    }
  ],
  "settings": []
}
```

接下來逐一介紹一下各個欄位的含義：

- `name` 和 `author`：Gopeed 中會使用 `<author>@<name>` 作為擴充功能的 ID，填寫 `author` 後能確保不容易和別的擴充功能重名導致被覆蓋安裝，所以強烈建議填寫 `author` 欄位。
- `title` 和 `description`：擴充功能的標題和描述。
- `icon`：擴充功能圖示，填寫相對路徑，範例：`icon.png`。
- `version`：擴充功能版本，使用 semver 規範，擴充更新時是基於此欄位進行比較的，所以請確保版本號是符合規範的。
- `homepage`：擴充功能首頁，範例：`https://gopeed.com`。
- `repository`：擴充功能所屬 git 存放庫位址，Gopeed 擴充功能依賴 `git` 來實現去中心化的擴充管理，所以如果你的擴充功能需要被使用者安裝和更新，那麼就必須將擴充原始碼託管到遠端 git 存放庫中，並設定此欄位

  範例：

  ```json
  {
    "url": "https://github.com/gopeed/gopeed-extension-demo"
  }
  ```

  如果一個 git 存放庫中包含多個擴充專案，可以透過 `directory` 屬性指定子目錄，範例：

  ```json
  {
    "url": "https://github.com/GopeedLab/gopeed-extension-samples",
    "directory": "github-contributor-avatars-sample"
  }
  ```

  在 Gopeed 安裝時，需要透過 `#` 進行分隔，即安裝連結為 `https://github.com/GopeedLab/gopeed-extension-samples#github-contributor-avatars-sample`。

- `scripts`：`敲重點！` 這裡是 Gopeed 擴充功能觸發事件的設定。

  範例專案中設定的 `onResolve` 事件會在解析任務時觸發，透過 `match.urls` 欄位來比對任務建立的 url，如果比對成功，則會執行 `entry` 欄位指定的腳本檔案。

  > 比對規則和 Chrome 擴充功能的比對規則一致，可以參考[這裡](https://developer.chrome.com/docs/extensions/mv3/match_patterns/)

- `settings`：擴充設定項，透過設定宣告可以在 Gopeed 中產生對應的設定介面，提供使用者自訂設定，例如自訂 `Cookie`、自訂 `User-Agent` 等等，範例：

  ```json
  [
    {
      "name": "ua",
      "title": "User-Agent",
      "description": "自訂 User-Agent",
      "type": "string",
      "value": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko)"
    }
  ]
  ```

  - `name`：設定項名稱，必填。
  - `title`：設定項標題，必填。
  - `description`：設定項描述，選填。
  - `type`：設定項類型，可選值：`string`、`number`、`boolean`。
  - `value`：設定項預設值，選填。

## 腳本編寫

上一節中，我們已經知道如何設定清單了，接下來就來介紹一下擴充腳本的編寫。

### 執行環境

Gopeed 擴充腳本引擎是由 [goja](https://github.com/dop251/goja) 來實現的，它是一個純 Go 編寫的 JavaScript 直譯器，但是由於 goja 只是一個純 js 執行環境，所以 `瀏覽器` 和 `node.js` 的 API 都是不支援的，目前 `Gopeed` 實作了 `XMLHttpRequest`、`fetch` 兩個 API，意味著你可以透過這兩個 API 或基於它們的第三方函式庫來實現網路請求，例如 `axios`、`superagent` 等等。

另外需要注意一點，goja 原生支援大部分 `es6+` 的語法，但是極少部分語法是不支援的，例如 `async generator`，不過沒關係，透過腳手架建立的專案中已經設定好了 `babel`，你可以愉快的使用最新的 es 語法，腳本最終會被編譯成 `es5` 語法。

### 範例腳本解析

每當事件觸發時，會執行 `entry` 欄位指定的腳本檔案，範例專案中的腳本檔案如下：

```js
gopeed.events.onResolve((ctx) => {
  ctx.res = {
    name: "example",
    files: [
      {
        name: "index.html",
        req: {
          url: "https://example.com",
        },
      },
    ],
  };
});
```

接下來，我們逐一介紹一下腳本的內容：

- `gopeed.events.onResolve`：這裡是註冊 `onResolve` 事件，方法裡面就是擴充功能的具體邏輯了。
- `ctx`：事件上下文，包含了目前事件的一些資訊，在 `onResolve` 事件中，`ctx` 包含了以下欄位：
  - `req`：請求資訊，包含了資源的 url、headers 等等。
  - `res`：回應資訊，腳本需要將解析出的檔案清單賦值給 `ctx.res`，Gopeed 會根據裡面回傳的檔案清單進行下載。

簡而言之就是需要在 `onResolve` 回呼函式中，根據 `ctx.req` 裡的請求資訊，解析出需要下載的檔案清單並賦值給 `ctx.res` 即可。

> 關於 `ctx` 相關型別定義與擴充原始碼，可以參考[官方 Gopeed JS 存放庫](https://github.com/GopeedLab/gopeed-js)。
